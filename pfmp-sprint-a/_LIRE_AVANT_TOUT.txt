================================================================
  PFMP Pilot AI — Sprint A (Vercel + Supabase + skeleton)
================================================================

CE QUE CONTIENT CE ZIP :

Ce ZIP reproduit la structure exacte de TON repo cloné PFMP-PiLot-V1.0.
Il faut juste l'extraire PAR-DESSUS ton dossier de repo, en
ÉCRASANT les fichiers existants quand demandé.

================================================================
  COMMENT L'INSTALLER (5 minutes)
================================================================

1. EXTRAIRE LE ZIP
   Clic droit sur ce ZIP → "Extraire tout..." → choisir un dossier
   temporaire (ex: Bureau/sprint-a/).

2. COPIER LE CONTENU PAR-DESSUS LE REPO
   Ouvre le dossier extrait. Sélectionne TOUT (Ctrl+A) sauf ce
   fichier _LIRE_AVANT_TOUT.txt.
   Glisse-dépose dans ton dossier PFMP-PiLot-V1.0.
   Quand Windows demande "Remplacer les fichiers ?", clique OUI
   POUR TOUT.

3. SUPPRIMER LES 2 FICHIERS OBSOLÈTES
   Dans ton dossier PFMP-PiLot-V1.0, supprime :
   - netlify.toml (à la racine)
   - supabase/schema.sql (s'il existe)

4. RECOMPILER
   Ouvre un terminal dans PFMP-PiLot-V1.0 (ou utilise le terminal
   intégré de GitHub Desktop : Repository → Open in Terminal).
   Tape :
       npm install
       npm run typecheck
       npm run build
   Tu dois voir "✓ built in Xs" à la fin sans erreur.

5. PUSHER VIA GITHUB DESKTOP
   Ouvre GitHub Desktop. Tu dois voir ~15 fichiers modifiés/créés.
   Dans "Summary", tape :
       feat: Supabase RLS + Vercel skeleton + multi-tenant (Sprint A)
   Clique "Commit to main", puis "Push origin".

6. VERCEL VA REDÉPLOYER AUTOMATIQUEMENT
   Va sur ton dashboard Vercel → Deployments → tu vois un nouveau
   build en cours. Attends 1-2 min → "Ready".
   Visite l'URL : le site continue de marcher en mode démo
   (RIEN NE DOIT CASSER).
