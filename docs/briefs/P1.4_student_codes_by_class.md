# P1.4 - Codes eleves par classe

> Brief placeholder - a detailler apres P1.3
> Date : 11 mai 2026
> Depend de : P1.3 import SIECLE classes + eleves

## Objectif fonctionnel

Un admin/DDFPT ou professeur principal genere les acces eleves d'une classe sans utiliser les emails eleves.

Chaque eleve recoit un code personnel et, si possible, un QR code. Le professeur peut imprimer une fiche classe.

## Principe produit

- Pas de code partage par classe.
- Un code unique par eleve.
- Code revocable/reinitialisable.
- Generation en masse depuis une classe.
- L'eleve entre plus tard via le portail "Eleve" avec son code.

## Schema probable

Une migration sera probablement necessaire :

- `student_access_codes`
  - `id`
  - `establishment_id`
  - `student_id`
  - `code_hash`
  - `code_hint`
  - `status`
  - `expires_at`
  - `last_used_at`
  - `created_by`
  - `created_at`
  - `revoked_at`

Les codes en clair ne doivent pas etre stockes. Ils sont affiches/imprimables uniquement au moment de la generation.

## UI cible

Depuis une classe :

- bouton "Generer les codes eleves" ;
- table des eleves avec statut code : absent / actif / revoque ;
- bouton "Reinitialiser" par eleve ;
- bouton "Telecharger fiche classe" ;
- bouton "Imprimer QR codes".

## Hors perimetre

- Connexion eleve effective.
- Espace eleve complet.
- Signature eleve.

Ces elements arrivent en P1.5/P1.6.
