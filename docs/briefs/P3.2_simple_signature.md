# P3.2 — Signature simple côté tuteur

> **Brief Codex** — workflow signature
> **Auteur :** Claude (orchestration)
> **Date :** 7 mai 2026
> **Statut :** À démarrer
> **Dépend de :** P3.1 mergé (token tuteur fonctionnel)

---

## Salut Codex

Sprint qui transforme le portail tuteur en **outil de signature**.
Le tuteur peut maintenant **valider la convention en ligne** depuis
son téléphone ou son ordinateur, sans imprimer/scanner/renvoyer.

⚠️ Signature **simple** (pas eIDAS qualifiée). Lis attentivement le
**Module 15 — Niveaux de signature** dans `docs/VISION.md`. Et
respecte **strictement** la contrainte juridique : ne jamais dire
qu'un scan ou une signature simple est qualifiée.

---

## Objectif fonctionnel

Le tuteur, sur la page `/sign/:token` (P3.1) :

1. Voit la fiche stage limitée (déjà fait en P3.1)
2. Voit maintenant un **bloc de signature** :
   - Case à cocher : "J'atteste avoir lu la convention et accepté les
     termes du stage"
   - Champ "Nom et prénom du signataire" (pré-rempli avec le tuteur)
   - Champ "Fonction" (libre)
   - **Pad de signature manuscrite** (avec souris/doigt sur mobile)
   OU bouton "Signer avec une signature dactylographiée"
3. Bouton "Signer la convention" (désactivé tant que la case n'est pas
   cochée)
4. Au clic :
   - Le système enregistre la signature
   - Calcule un hash du document signé
   - Marque le token comme `signed_at`
   - Génère un **document de preuve** (PDF avec mentions de la
     signature : qui, quand, IP, UA, méthode = "signature simple par
     lien sécurisé")
5. Confirmation : "Convention signée. Le professeur référent et la
   direction du lycée ont été notifiés."

## Contraintes non-négociables

### ⚠️⚠️⚠️ Mentions juridiques OBLIGATOIRES

Sur la page de signature, **affichage clair** :

> ℹ️ **Information légale**
>
> Cette signature est une **signature électronique simple** (article 1366
> du Code civil français). Elle a la même valeur qu'une signature
> manuscrite **entre les parties qui l'ont acceptée**. Elle n'est pas
> une signature électronique avancée ou qualifiée au sens du règlement
> eIDAS.
>
> Les informations suivantes sont collectées à des fins probatoires :
> votre adresse IP, votre navigateur, la date et l'heure exacte, et le
> hash cryptographique du document signé. Ces données sont conservées
> conformément à notre politique de confidentialité.

→ Texte fixe, pas modifiable par l'admin lycée. C'est une **mention
légale produit** que BraveHeart porte juridiquement.

### 🛡️ Single-use

Une fois signé, le token est **brûlé** : `signed_at` est mis, et toute
re-tentative de signature retourne une erreur "Convention déjà
signée".

### 🛡️ Pas de modification rétroactive

Une fois signée, **la convention ne peut plus être modifiée** :
- L'admin lycée ne peut pas changer le PDF
- Le tuteur ne peut pas re-signer
- Le hash du PDF est figé

Si une correction est nécessaire (erreur, changement de tuteur,
etc.) : il faut **invalider** la convention actuelle (`revoked_at`)
et en générer une nouvelle (nouveau token, nouvelle signature).

### 🛡️ Hash du document signé

Au moment de la signature, calcul du SHA-256 du PDF actuel et
sauvegarde dans `tutor_access_tokens.signed_document_hash`.

Permet de prouver plus tard : "Voici le PDF qui a été présenté au
tuteur, voici sa signature, le hash matche."

### 🛡️ Document de preuve

Génération automatique d'un PDF de **preuve** qui contient :
- Le PDF original signé (en page 1+)
- Une **page d'attestation** (en dernière page) avec :
  - Identité du signataire (nom, fonction, email)
  - Date et heure exacte de signature
  - IP et UA du signataire
  - Hash SHA-256 du document
  - Méthode de signature : "signature électronique simple"
  - QR code optionnel pointant vers une URL de vérification (futur)

Stocké dans bucket `proof-files`.

### Fichier 2 : Edge Function `supabase/functions/sign-tutor-document/index.ts`

```typescript
serve(async (req) => {
  const {
    token,
    signerName,
    signerFunction,
    signatureMethod,  // 'simple_typed' | 'simple_drawn'
    signatureData,    // base64 si drawn, null si typed
    accepted,         // bool : case à cocher
  } = await req.json()

  if (!accepted) {
    return new Response('You must accept the terms', { status: 400 })
  }

  if (!signerName || signerName.trim().length < 2) {
    return new Response('Signer name required', { status: 400 })
  }

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!,
  )

  const tokenHash = await hashToken(token)

  const { data: tokenRow } = await supabase
    .from('tutor_access_tokens')
    .select('*, placement:placements(*, student:students(*), company:companies(*))')
    .eq('token_hash', tokenHash)
    .is('revoked_at', null)
    .single()

  if (!tokenRow) return new Response('Invalid token', { status: 404 })
  if (tokenRow.signed_at) return new Response('Already signed', { status: 409 })
  if (new Date(tokenRow.expires_at) < new Date())
    return new Response('Token expired', { status: 410 })

  // 1. Récupérer le PDF de la convention
  const { data: conventionDoc } = await supabase
    .from('documents')
    .select('storage_path, content_hash')
    .eq('placement_id', tokenRow.placement_id)
    .eq('type', 'convention')
    .order('generated_at', { ascending: false })
    .limit(1)
    .single()

  if (!conventionDoc) return new Response('No convention generated', { status: 412 })

  // 2. Hash le PDF actuel (vérifier intégrité)
  const { data: pdfBlob } = await supabase.storage
    .from('generated-pdfs')
    .download(conventionDoc.storage_path)
  const pdfHash = await sha256(await pdfBlob.arrayBuffer())

  // 3. Générer le PDF de preuve (PDF original + page d'attestation)
  const ip = req.headers.get('x-forwarded-for') ?? 'unknown'
  const ua = req.headers.get('user-agent') ?? 'unknown'

  const proofPdf = await generateProofPdf({
    originalPdfBuffer: await pdfBlob.arrayBuffer(),
    metadata: {
      signerName,
      signerFunction,
      signedAt: new Date().toISOString(),
      ip, ua,
      documentHash: pdfHash,
      method: signatureMethod,
      placementId: tokenRow.placement_id,
      studentName: `${tokenRow.placement.student.first_name} ${tokenRow.placement.student.last_name}`,
    },
    signatureImageBase64: signatureData,
  })

  // 4. Storage proof-files
  const proofPath = `${tokenRow.establishment_id}/${tokenRow.placement_id}/proof_convention_${Date.now()}.pdf`
  await supabase.storage.from('proof-files').upload(proofPath, proofPdf)

  // 5. UPDATE token
  await supabase
    .from('tutor_access_tokens')
    .update({
      signed_at: new Date().toISOString(),
      signed_document_hash: pdfHash,
      signature_method: signatureMethod,
      signer_name: signerName,
      signer_function: signerFunction,
      signer_signature_data: signatureData,
      signer_ip: ip,
      signer_user_agent: ua,
      proof_file_path: proofPath,
    })
    .eq('id', tokenRow.id)

  // 6. Audit log
  await supabase.from('audit_logs').insert({
    establishment_id: tokenRow.establishment_id,
    action: 'tutor_signed_convention',
    target_type: 'tutor_access_token',
    target_id: tokenRow.id,
    metadata: {
      placement_id: tokenRow.placement_id,
      signer_name: signerName,
      signature_method: signatureMethod,
      document_hash: pdfHash,
      proof_path: proofPath,
    }
  })

  // 7. Notifier le prof référent et l'admin (email_queue)
  await supabase.from('email_queue').insert({
    to_email: '...',  // récupérer email du prof référent
    template_key: 'tutor_signed_notification',
    template_data: { ... }
  })

  return Response.json({
    success: true,
    signedAt: new Date().toISOString(),
    proofUrl: '...'  // signed URL du PDF preuve
  })
})
```

### Fichier 5 : `src/lib/pdf/proofGenerator.ts`

Génère le PDF de preuve (concat original + page attestation).

Utilise `pdf-lib` (lib lightweight, fonctionne en Edge Function) :

```typescript
import { PDFDocument, rgb, StandardFonts } from 'pdf-lib'

export async function generateProofPdf(input: {
  originalPdfBuffer: ArrayBuffer
  metadata: ProofMetadata
  signatureImageBase64?: string
}): Promise<Uint8Array> {
  const proofDoc = await PDFDocument.create()

  // 1. Copier toutes les pages du PDF original
  const originalDoc = await PDFDocument.load(input.originalPdfBuffer)
  const originalPages = await proofDoc.copyPages(originalDoc, originalDoc.getPageIndices())
  originalPages.forEach(p => proofDoc.addPage(p))

  // 2. Ajouter une page d'attestation
  const attestationPage = proofDoc.addPage()
  const { width, height } = attestationPage.getSize()
  const helvetica = await proofDoc.embedFont(StandardFonts.Helvetica)
  const helveticaBold = await proofDoc.embedFont(StandardFonts.HelveticaBold)

  let y = height - 80
  attestationPage.drawText('ATTESTATION DE SIGNATURE ÉLECTRONIQUE SIMPLE', {
    x: 50, y, size: 16, font: helveticaBold,
  })
  y -= 40

  const lines = [
    `Document signé : Convention de stage`,
    `Stagiaire : ${input.metadata.studentName}`,
    `Signataire : ${input.metadata.signerName}`,
    `Fonction : ${input.metadata.signerFunction || '—'}`,
    `Date et heure : ${new Date(input.metadata.signedAt).toLocaleString('fr-FR')}`,
    `Méthode : Signature électronique simple par lien sécurisé`,
    `Adresse IP : ${input.metadata.ip}`,
    `User-Agent : ${input.metadata.ua}`,
    `Hash SHA-256 du document : ${input.metadata.documentHash}`,
    ``,
    `Cette signature est une signature électronique simple au sens de`,
    `l'article 1366 du Code civil français. Elle n'est pas une`,
    `signature électronique avancée ou qualifiée au sens du règlement`,
    `eIDAS (UE) n° 910/2014.`,
  ]

  for (const line of lines) {
    attestationPage.drawText(line, { x: 50, y, size: 11, font: helvetica })
    y -= 18
  }

  // 3. Si signature manuscrite : embed image
  if (input.signatureImageBase64) {
    const sigBytes = base64ToBytes(input.signatureImageBase64)
    const sigImage = await proofDoc.embedPng(sigBytes)
    const sigDims = sigImage.scale(0.3)
    attestationPage.drawImage(sigImage, {
      x: 50, y: y - 100, width: sigDims.width, height: sigDims.height,
    })
  }

  return await proofDoc.save()
}
```

## ⚠️ Note sur la conformité juridique

BraveHeart **doit** vérifier avec un juriste avant le 1er client
payant. La signature simple a une **valeur probante** mais peut être
**contestée**. Pour des conventions de stage en lycée :
- En général : signature simple suffit (faible enjeu juridique)
- En cas de contentieux : juge peut demander la qualifiée

Le service "Premium IA" pourrait inclure l'option **eIDAS qualifiée**
via prestataire (P4.4). Pour P3.2, la simple est suffisante pour
démarrer le pilote.
