## Salut Codex

Bienvenue dans la phase **P2 — Intelligence référentielle**. C'est le
cœur stratégique du produit. Sans ces 7 sprints, PFMP Pilot reste un
"Pronote pour les PFMP". Avec, il devient un **vrai SaaS B2B
intelligent et vendable**.

Ce sprint P2.1 pose les **fondations data**. Pas d'IA, pas d'UI
complexe — juste un schema solide qui supporte toute la règle d'or
référentielle. Tout le reste de P2 dépend de ce sprint.

Lis attentivement la section "Modèle de données" de `docs/VISION.md`
avant de commencer.

---

## Objectif fonctionnel

À la fin de ce sprint :

1. La DB contient deux nouvelles tables :
   - `formation_referential` (référentiels nationaux et locaux)
   - `establishment_formation_setup` (mapping tenant ↔ référentiel ↔
     templates)
2. La DB contient un seed initial de **4 référentiels nationaux** :
   - Bac Pro MELEC (3 ans)
   - Bac Pro PCEPC (3 ans)
   - CAP AQE (2 ans)
   - CAP ETL (2 ans)
3. RLS configurée :
   - Référentiels `national` lisibles par tous les users authentifiés
   - Référentiels `establishment` lisibles uniquement par leur tenant
   - Modification réservée superadmin pour les nationaux, admin/ddfpt
     pour les locaux
4. Triggers anti-cross-tenant comme pour les autres tables P0
5. La table `classes` est augmentée d'une colonne
   `formation_referential_id`

---

## Contraintes non-négociables

### 🛡️ Migration idempotente

La migration doit pouvoir être rejouée sans casser la prod existante.
Utilise `if not exists`, `if not exists` sur les indexes, et des
guards sur les `add column`.

### 🛡️ RLS strict comme les sprints précédents

- Pas de policy permissive `using (true)`
- Vérification via `current_establishment_id()` JWT-aware existante
- Trigger anti-tenant-change si `establishment_id` change

### 🛡️ Compatibilité avec données existantes

Les classes existantes (créées en P1.1/P1.2) **n'ont pas** de
`formation_referential_id` au moment de cette migration. C'est OK —
la colonne est nullable. Le rattachement viendra en **P2.3** via UI.

### 🛡️ Pas d'import des AP/compétences hardcodés

Le seed initial des 4 référentiels nationaux doit avoir des **grilles
vides ou minimales** (`attitudes_grid: '[]'::jsonb`,
`competences_grid: '[]'::jsonb`). C'est l'**Analyseur IA (P2.2)** qui
les remplira en analysant les .docx Blaïse Cendrars.

Erreur à éviter : ne pas inventer les compétences MELEC à la main —
c'est précisément le boulot de l'IA en P2.2 et de la validation
humaine.

## Schema détaillé

### Table `formation_referential`

```sql
create table if not exists formation_referential (
  id uuid primary key default gen_random_uuid(),

  -- Origine
  source text not null check (source in ('national', 'establishment')),
  parent_id uuid references formation_referential(id) on delete restrict,
  establishment_id uuid references establishments(id) on delete cascade,

  -- Identité
  formation_code text not null,
  formation_label text not null,
  year_level smallint not null check (year_level between 1 and 5),

  -- Durée et structure
  total_pfmp_weeks smallint,
  periods_per_year smallint default 2,
  default_period_dates jsonb default '[]'::jsonb,
  -- ex: [{"name":"P1","start_month":3,"start_day":17,"duration_weeks":2}]

  -- Grilles (remplies par l'analyseur IA en P2.2)
  attitudes_grid jsonb not null default '[]'::jsonb,
  -- [{"code":"AP1","label":"Intérêt","levels":[{"value":1,"label":"Aucun"}, ...]}]
  competences_grid jsonb not null default '[]'::jsonb,
  -- [{"code":"C1.1","label":"Préparer une opération","indicators":[...]}]

  -- Méta
  notes text,
  active boolean not null default true,
  created_by uuid references profiles(id) on delete set null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  -- Cohérence
  constraint chk_source_consistency check (
    (source = 'national' and establishment_id is null and parent_id is null)
    or
    (source = 'establishment' and establishment_id is not null)
  ),
  constraint uq_referential_per_source unique (source, formation_code, year_level, establishment_id)
);

create index if not exists idx_referential_source on formation_referential(source);
create index if not exists idx_referential_estab on formation_referential(establishment_id);
create index if not exists idx_referential_code on formation_referential(formation_code, year_level);
create index if not exists idx_referential_parent on formation_referential(parent_id);
```

**Logique source/parent :**
- `national` : pas de tenant, pas de parent. Visible par tous.
- `establishment` : tenant obligatoire. Si `parent_id` non null, hérite
  d'un référentiel national (overrides). Si `parent_id` null, c'est un
  référentiel **100% local** (cas rare : lycée avec formation maison).

### Table `establishment_formation_setup`

```sql
create table if not exists establishment_formation_setup (
  id uuid primary key default gen_random_uuid(),
  establishment_id uuid not null references establishments(id) on delete cascade,
  formation_referential_id uuid not null references formation_referential(id) on delete restrict,

  -- Templates de documents (les .docx originaux du lycée)
  -- Stockés dans Supabase Storage `templates` bucket
  convention_template_path text,
  livret_template_path text,
  attestation_template_path text,

  -- Customisations spécifiques (texte des mentions légales adaptées par exemple)
  customizations_json jsonb default '{}'::jsonb,

  active boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  unique (establishment_id, formation_referential_id)
);

create index if not exists idx_setup_estab on establishment_formation_setup(establishment_id);
create index if not exists idx_setup_referential on establishment_formation_setup(formation_referential_id);
```

### Augmentation de `classes`

```sql
alter table classes
  add column if not exists formation_referential_id uuid
    references formation_referential(id) on delete set null;

create index if not exists idx_classes_referential on classes(formation_referential_id);
```

### RLS

```sql
alter table formation_referential enable row level security;
alter table establishment_formation_setup enable row level security;

-- formation_referential

-- Lecture des nationaux : tout user authentifié
create policy "national_referentials_readable_by_all"
  on formation_referential
  for select
  using (
    source = 'national'
    or establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  );

-- Modification des nationaux : superadmin only
create policy "national_referentials_writable_by_superadmin"
  on formation_referential
  for all
  using (
    source = 'national'
    and current_role_is_superadmin()
  )
  with check (
    source = 'national'
    and current_role_is_superadmin()
  );

-- Modification des locaux : admin/ddfpt du tenant
create policy "local_referentials_writable_by_estab_admin"
  on formation_referential
  for all
  using (
    source = 'establishment'
    and establishment_id = current_establishment_id()
    and current_role_in('admin', 'ddfpt')
  )
  with check (
    source = 'establishment'
    and establishment_id = current_establishment_id()
    and current_role_in('admin', 'ddfpt')
  );

-- establishment_formation_setup

create policy "setup_isolation"
  on establishment_formation_setup
  for all
  using (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  )
  with check (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  );
```

**Note sur les helpers :**
- `current_establishment_id()` existe déjà (Sprint 2)
- `current_role_is_superadmin()` à créer si pas existant
- `current_role_in(...variadic text)` à créer si pas existant
