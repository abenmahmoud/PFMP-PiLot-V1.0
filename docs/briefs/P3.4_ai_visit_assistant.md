## Salut Codex

Sprint **gain de temps prof**. Aujourd'hui, un prof qui rentre d'une
visite en entreprise tape péniblement son CR sur son téléphone.
Demain, il dicte 3 phrases courtes et l'IA lui génère un CR
professionnel propre, qu'il valide ou ajuste avant de soumettre.

C'est **l'option Premium IA** facturée plus cher dans le modèle
commercial.

Lis attentivement le **Module 11 — Assistant IA professeur** dans
`docs/VISION.md`.

---

## Objectif fonctionnel

Sur le formulaire de visite (`/visits/new`) :

1. Le prof remplit les champs **factuels** :
   - Date, type contact, élève présent, tuteur rencontré
2. Pour les **champs textes longs** (conditions, activités, points
   positifs, difficultés, remarques) :
   - Le prof saisit des **notes courtes** (ou utilise la dictée vocale
     du téléphone)
   - Bouton "✨ Reformuler avec l'IA" à côté du champ
3. Au clic, l'IA :
   - Reçoit les notes brutes
   - Reçoit le **contexte métier** (formation, métier, niveau)
   - Génère un brouillon professionnel
   - Le résultat apparaît dans le champ (remplaçable, modifiable)
4. **Le prof valide toujours** avant de soumettre la visite

Et un mode **plus avancé** :

5. Bouton "✨ Générer le CR complet" (en bas du formulaire)
6. L'IA prend tous les champs notes + facts → génère un CR cohérent
   pour tous les champs longs en une seule fois

## Contraintes non-négociables

### 🛡️ Ne jamais inventer

Le prompt doit être **strict** :

> Tu reformules les notes du professeur en français professionnel
> scolaire. **N'invente AUCUN fait** qui n'est pas dans les notes
> brutes. Ne suppose rien. Si tu ne sais pas, dis "à compléter par le
> professeur".

Si l'IA invente quelque chose, c'est une **régression critique** :
elle pourrait écrire que l'élève a fait des activités qu'il n'a pas
faites → dossier de preuve compromis.

### 🛡️ Brouillon uniquement

Le prof voit toujours le résultat **avant** de soumettre. Aucune
soumission auto.

### 🛡️ Logs ai_interactions

Chaque appel IA :
```
{
  user_id, establishment_id, assistant_type: 'visit_report',
  request: { notes_length, target_field, student_id, formation_code },
  response: { tokens_in, tokens_out, length_chars },
  duration_ms, model_used
}
```

### 🛡️ Coût contrôlé

- Modèle : **Claude Haiku** (assez bon, peu cher)
- Limite : 50 appels/jour/prof, 500/jour/tenant
- Si dépassé : message "Limite quotidienne atteinte. Réessayez demain
  ou contactez votre administrateur."

### 🛡️ RLS et permissions

Edge Function vérifie que l'user est bien un référent du tenant ET
qu'il a accès à l'élève (via `teacher_assignments`).

```typescript
  // 5. Appel Anthropic Haiku
  // 6. Log ai_interactions
  // 7. Retourner texte généré
})
```

### Prompt template

```typescript
function buildPrompt(input: RewriteInput): string {
  return `Tu es un assistant qui aide un professeur à rédiger un compte
rendu de visite de stage en milieu professionnel (PFMP).

CONTEXTE :
- Formation de l'élève : ${input.formationLabel} année ${input.yearLevel}
- Métier : ${input.professionalFamily}
- Entreprise : ${input.companyName}
- Type de contact : ${input.contactType}

CHAMP À REMPLIR : ${input.targetFieldLabel}
(par exemple : "Activités réalisées par l'élève", "Points positifs",
"Difficultés rencontrées", "Remarques du tuteur", etc.)

NOTES BRUTES DU PROFESSEUR :
${input.rawNotes}

CONSIGNES STRICTES :
- Reformule en français professionnel, ton neutre et scolaire
- N'invente AUCUN fait qui n'est pas dans les notes brutes
- Reste factuel, pas d'interprétation excessive
- Garde toutes les nuances importantes (positives ET négatives)
- Maximum 5-6 phrases pour ce champ
- Si les notes sont trop laconiques pour reformuler proprement,
  réponds : "Notes trop courtes pour être reformulées. Veuillez
  développer."

Réponse (texte uniquement, pas de préambule, pas de guillemets) :`
}
```
