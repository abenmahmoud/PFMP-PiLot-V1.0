## Salut Codex

Sprint **conformité juridique**. Tu construis le **dossier de preuve**
complet qui permet à un lycée, en cas de contentieux ou d'audit, de
prouver l'historique d'une PFMP : qui a signé quoi, quand, comment.

C'est aussi ce qui permet à BraveHeart de **vendre** le produit avec
confiance : "regardez, vous pouvez exporter à tout moment un dossier
complet juridiquement solide".

---

## Objectif fonctionnel

L'admin / DDFPT / prof référent peut :

1. Aller sur la fiche d'un élève ou sur une période PFMP
2. Cliquer "Exporter le dossier de preuve"
3. Le système génère un **ZIP** contenant :
   - `convention_signed_proof.pdf` (PDF original + attestation
     signature P3.2)
   - `livret_pfmp.pdf` (livret rempli si dispo)
   - `attestation_stage.pdf` (attestation finale si dispo)
   - `visites/` (PDF de chaque CR de visite validé)
   - `audit_log.json` (toutes les actions liées à cet élève/période)
   - `manifest.json` (métadonnées du dossier : qui a généré, quand,
     pour qui, hashes de chaque fichier)
4. Téléchargement du ZIP via signed URL

---

## Contraintes non-négociables

### 🛡️ Cohérence des hashes

Chaque fichier dans le ZIP a son hash SHA-256 dans le `manifest.json`.
Permet de vérifier l'intégrité plus tard.

### 🛡️ Audit log filtré

Le `audit_log.json` ne contient **que les events liés à cet élève /
période / placement**. Pas tout l'audit log du tenant.

### 🛡️ Permissions

- Admin / DDFPT du tenant : peut exporter pour n'importe quel élève
  du tenant
- Prof référent : peut exporter uniquement pour ses élèves affectés
- Élève : pas accès (lecture seule de ses propres docs via son espace
  futur)

### 🛡️ Nommage cohérent

Pattern : `{nom_eleve}_{prenom_eleve}_{periode_code}_{date_export}.zip`

Exemple : `DUPONT_Marie_P1-2025_2025-04-15.zip`

### 🛡️ Pas de modification après export

Le manifest mentionne "État au {date_export}". Si l'élève fait un
nouveau stage plus tard, c'est un autre dossier de preuve.

## Architecture technique

### Edge Function `supabase/functions/generate-proof-bundle/index.ts`

```typescript
serve(async (req) => {
  // 1. Auth + permissions check (admin/ddfpt/referent du tenant + accès à l'élève)
  // 2. Charger toutes les données :
  //    - student + class + formation_referential
  //    - placement(s) actif(s) ou de la période
  //    - documents (convention, livret, attestation)
  //    - tutor_access_tokens (signature data)
  //    - visits validées
  //    - audit_logs filtrés (action concerne cet élève/placement)
  //    - email_queue (notifications envoyées)
  // 3. Pour chaque doc à inclure :
  //    - Télécharger depuis storage
  //    - Calculer le hash SHA-256
  // 4. Construire le manifest.json
  // 5. Construire l'audit_log.json filtré
  // 6. Zipper le tout (lib JSZip ou similaire)
  // 7. Stocker le ZIP dans bucket `proof-files` ou retourner direct
  // 8. Audit log de l'export
  // 9. Retourner signed URL
})
```

## Test obligatoire

Avec un élève Blaïse Cendrars qui a :
- Une convention signée par tuteur (P3.2)
- Un livret généré (P2.6)
- 2 visites validées (P0.7-P0.8)

1. Login DDFPT
2. Aller sur fiche de cet élève
3. Cliquer "Exporter le dossier de preuve"
4. Télécharger le ZIP
5. Décompresser, vérifier la structure :
```
DUPONT_Marie_P1-2025_2026-05-07.zip
├── manifest.json
├── audit_log.json
├── convention_signed_proof.pdf
├── livret_pfmp.pdf
└── visites/
    ├── visite_2025-03-20.pdf
    └── visite_2025-03-25.pdf
```
6. Vérifier les hashes (sha256sum vs manifest.json)
7. Tenter d'exporter le même élève en tant que prof référent qui ne
   l'a pas → erreur 403
8. Cleanup
