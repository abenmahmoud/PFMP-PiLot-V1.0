## Contexte

`src/routes/dashboard.tsx` est aujourd'hui 100% démo : il importe
`pfmpPeriods, students, visits, alerts, documents, activityLog,
companies, ESTABLISHMENT_ID` depuis `@/data/demo`.

Il calcule plein de KPIs côté client : élèves en stage, élèves sans
stage, visites faites, conventions manquantes, attestations manquantes,
familles métiers couvertes, charge professeurs, alertes.

Tout doit venir de Supabase, **filtré par tenant via RLS naturelle**
(le user loggé voit son tenant, le superadmin voit le tenant
sélectionné via le switcher → `current_establishment_id()` JWT-aware).

---

## Objectif fonctionnel

Un user DDFPT loggé en prod sur le dashboard voit :

1. Les KPIs de son établissement, calculés depuis Supabase
2. Si Supabase retourne 0 rows partout (tenant vide → cas Blaïse
   Cendrars au démarrage) → empty states clairs et actions "créer
   première classe" / "créer première période PFMP"
3. Le mode démo continue de fonctionner exactement comme avant

## Contraintes non-négociables

### 🛡️ Mode démo intact

- Le bridge `useCurrentUser` est déjà câblé
- Si `isDemoMode() === true`, dashboard utilise `data/demo` comme avant
- Si `isDemoMode() === false` ET user loggé, dashboard utilise Supabase

### 🛡️ Pas de calculs client si évitable

Pour les KPIs simples (count élèves, count visites), fais des **count
queries SQL** côté Supabase, pas un fetch all puis count en JS. Ça
scalera mieux à 100+ tenants.

Exception : pour les calculs cross-tables complexes (familles
métiers couvertes, charge profs), un fetch puis aggregate JS est OK
en P0.3, optimisable plus tard.

### 🛡️ RLS naturelle

Tu n'as **pas besoin** de filtrer par `establishment_id` dans tes
requêtes : la RLS le fait automatiquement via
`current_establishment_id()`. Tu écris :

```ts
const { count } = await sb.from('students').select('*', { count: 'exact', head: true })
```

Et la RLS filtre. Si tu mets un `.eq('establishment_id', xxx)` toi-même,
c'est redondant (sauf cas superadmin qui veut un tenant spécifique).

### 🛡️ Pattern hooks

Crée `src/services/dashboard.ts` (ou similaire) qui expose des fonctions
async typed. Pas de fetch direct dans le composant route.

### 🛡️ Pattern de loading

Pendant le fetch initial, affiche un skeleton (pas un spinner blanc).
Regarde si `src/components/ui/` a déjà un Skeleton.
