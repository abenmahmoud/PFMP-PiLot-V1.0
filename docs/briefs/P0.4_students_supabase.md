## Salut Codex

Sprint en suite logique de P0.3. On applique le même pattern (hook
service + lecture Supabase + empty states + démo intacte) aux pages
élèves. Plus rapide que P0.3 parce que tu connais maintenant la
structure.

---

## Objectif fonctionnel

1. `/students` (liste) lit Supabase, filtré par tenant via RLS
2. `/students/:id` (détail) lit Supabase, retourne 404 propre si élève
   inexistant ou autre tenant
3. `/students/index` (équivalent ?) à clarifier dans ton plan
4. Filtres / recherche fonctionnent (par classe, par statut stage, par
   prof référent)
5. Empty state propre

---

## Contraintes (rappel rapide)

- Mode démo intact
- RLS naturelle (pas de filter `establishment_id` redondant)
- Skeleton pendant fetch
- Pas de N+1
- typecheck + build verts
