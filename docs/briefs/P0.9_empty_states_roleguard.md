# P0.9 — Empty states + RoleGuard sur toutes les routes

> **Brief Codex** — sprint final P0
> **Auteur :** Claude (orchestration)
> **Date :** 7 mai 2026
> **Statut :** À démarrer
> **Dépend de :** P0.8 mergé

---

## Salut Codex

Dernier sprint P0. On polit le tour complet pour qu'un user prod puisse
naviguer **sans aucun crash, sans aucune page blanche, sans aucun
écran cassé**.

C'est le sprint qui rend le produit montrable à un client pilote.

---

## Objectif fonctionnel

À la fin de ce sprint :

1. Toutes les routes affichent un **empty state propre** quand
   Supabase retourne 0 rows
2. Toutes les routes ont un **RoleGuard** correct (un référent ne peut
   pas accéder à `/superadmin`, etc.)
3. Toutes les routes ont un **skeleton de chargement** sobre
4. Tous les liens de navigation sont **fonctionnels** (pas de 404)
5. Le parcours MVP étapes 7-12 (référent fait visite et valide CR) est
   **fluide en prod**

---

## Périmètre

### Étape 1 : Tour complet routes en mode prod vide

Login `test@pfmp-pilot.fr` (superadmin), naviguer dans l'ordre :

| Route | Comportement attendu |
|---|---|
| `/` | Redirige vers `/superadmin` (car superadmin) |
| `/superadmin` | Liste vide d'établissements → empty state avec CTA "Créer un établissement" |
| `/superadmin/establishments` | Idem |
| `/superadmin/ai` | Page placeholder OK |
| `/dashboard` | Si superadmin sans switch → empty + CTA pour switcher tenant. Si tenant switché vide → KPIs à 0 + empty states par bloc |
| `/students` | Empty state avec lien vers import |
| `/students/$id` | 404 propre si UUID inexistant |
| `/teachers` | Empty state |
| `/companies` | Empty state |
| `/pfmp-periods` | Empty state |
| `/assignments` | Empty state |
| `/my-students` | Empty state spécifique référent |
| `/visits/new` | Doit être bloqué si pas d'élèves |
| `/documents` | Empty state |
| `/alerts` | Empty state |
| `/settings` | Page accessible (paramétrage = P2 en vrai) |

Documente chaque cas dans le rapport.

### Étape 2 : RoleGuard

- Vérifier `src/components/RoleGuard.tsx` (existe)
- Brancher sur les routes sensibles :
  - `/superadmin/*` → require `superadmin`
  - `/assignments`, `/pfmp-periods` → require `admin` ou `ddfpt` ou
    `principal`
  - `/my-students` → require `referent`
- Comportement quand rôle KO : redirect ou page d'erreur explicite,
  pas crash

### Étape 3 : Skeletons

Vérifier qu'il y a un skeleton (ou loading state propre) sur :
- dashboard
- students (liste)
- companies (liste)
- my-students
- visits.$id

### Étape 4 : Liens navigation

Cliquer sur tous les items du Sidebar et de Topbar. Aucun ne doit :
- Rediriger vers une 404
- Afficher une page blanche
- Crasher
