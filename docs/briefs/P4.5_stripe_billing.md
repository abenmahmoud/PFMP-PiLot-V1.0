## Salut Codex

Sprint **clôture roadmap**. Tu intègres Stripe pour facturer les
lycées clients. Sans ce sprint, BraveHeart facture à la main par
virement bancaire — ça marche pour 2-3 lycées, pas pour 50.

C'est aussi ce qui débloque la **vente en self-service** à terme
(formulaire `/contact` peut basculer vers paiement direct si le lycée
est en autonomie totale).

---

## Objectif fonctionnel

### Côté commercial / superadmin

1. BraveHeart configure les **produits Stripe** :
   - Plan Standard (sans IA)
   - Plan Premium IA
   - Add-on Mise en place (forfait one-time)
   - Add-on Signatures qualifiées (à l'usage : facturation par
     compteur, ou par tranche)
2. À la fin de l'onboarding (P4.3), génère un **lien Stripe
   Checkout** envoyé au comptable du lycée (UGAP, agent comptable,
   etc.)
3. Le client paye en CB ou par virement (Stripe gère)
4. Webhook Stripe → `establishments.subscription_status = 'active'`
5. Tenant pleinement actif

### Côté tenant client

1. Page `/settings/facturation` (admin lycée) qui montre :
   - Plan actuel
   - Date de prochain renouvellement
   - Historique des factures (PDF téléchargeables)
   - Bouton "Mettre à jour mes infos de paiement" (Stripe Customer
     Portal)
2. **Pas de self-checkout** dans cette V1 (tous les lycées passent par
   BraveHeart pour l'onboarding)

## Contraintes non-négociables

### 🛡️ Pas de carte bleue stockée

Stripe gère TOUT le PCI-DSS. PFMP Pilot ne stocke jamais de numéro de
CB. Les utilisateurs interagissent avec :
- Stripe Checkout (achat)
- Stripe Customer Portal (modification CB / coordonnées)

Tout passe par les pages hébergées Stripe.

### 🛡️ Webhooks signés

Tous les webhooks Stripe **doivent** vérifier la signature
`stripe-signature` (HMAC SHA-256) avec le secret webhook. Sinon
rejet 401.

### 🛡️ Idempotence

Les events Stripe peuvent être livrés plusieurs fois. Vérifier l'event
ID avant de traiter.

### 🛡️ Synchronisation `establishments.subscription_status`

Statuts possibles :
- `trialing` — pendant les 90 jours d'essai
- `active` — abonnement payant en cours
- `past_due` — paiement échoué, accès maintenu pendant grace period
- `canceled` — annulé (lecture seule pendant 30j puis désactivation)
- `paused` — suspendu (BraveHeart ou admin)

Et synchroniser :
- `establishments.stripe_customer_id`
- `establishments.stripe_subscription_id`
- `establishments.subscription_current_period_end`

### 🛡️ Mode démo

En démo : pas d'appel Stripe. Tout mocké.

## Schema additionnel

### Migration `supabase/migrations/0016_stripe_billing.sql`

```sql
-- Augmentation establishments
alter table establishments
  add column if not exists stripe_customer_id text,
  add column if not exists stripe_subscription_id text,
  add column if not exists subscription_status text default 'trialing'
    check (subscription_status in ('trialing', 'active', 'past_due', 'canceled', 'paused')),
  add column if not exists subscription_plan text
    check (subscription_plan in ('standard', 'premium_ia')),
  add column if not exists subscription_current_period_start timestamptz,
  add column if not exists subscription_current_period_end timestamptz,
  add column if not exists subscription_trial_ends_at timestamptz,
  add column if not exists billing_email text;

create index if not exists idx_establishments_stripe_customer
  on establishments(stripe_customer_id);
create index if not exists idx_establishments_subscription_status
  on establishments(subscription_status);

-- Table invoices (cache local des factures Stripe)
create table if not exists invoices (
  id uuid primary key default gen_random_uuid(),
  establishment_id uuid not null references establishments(id) on delete cascade,
  stripe_invoice_id text not null unique,
  amount_paid_eur numeric(10, 2) not null,
  currency text not null default 'eur',
  status text not null,  -- paid, open, void, uncollectible
  hosted_invoice_url text,
  invoice_pdf_url text,
  paid_at timestamptz,
  created_at timestamptz not null default now()
);

create index if not exists idx_invoices_establishment on invoices(establishment_id);

alter table invoices enable row level security;
create policy "invoices_tenant_read" on invoices
  for select
  using (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  );

-- Pas de policy d'INSERT/UPDATE — uniquement Edge Functions avec service_role

-- Table stripe_events (idempotence webhook)
create table if not exists stripe_events (
  id uuid primary key default gen_random_uuid(),
  stripe_event_id text not null unique,
  event_type text not null,
  payload jsonb not null,
  processed_at timestamptz,
  error_message text,
  received_at timestamptz not null default now()
);

create index if not exists idx_stripe_events_processed on stripe_events(processed_at);

alter table stripe_events enable row level security;
create policy "stripe_events_superadmin" on stripe_events
  for select using (current_role_is_superadmin());
```
