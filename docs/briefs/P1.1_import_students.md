## Objectif fonctionnel

L'admin (ou le DDFPT) peut :

1. Aller sur `/import/students` (route nouvelle ou onglet sur
   `/students`)
2. Drag & drop un CSV
3. Voir un **mapping intelligent** : "votre colonne `NOM` → champ
   `last_name`, votre colonne `PRENOM` → `first_name`, votre colonne
   `DIV.` → `class_name`, ..."
4. Modifier le mapping si besoin
5. Voir une **prévisualisation** des 5 premières lignes mappées
6. Voir les **erreurs détectées** (lignes mal formées, doublons, etc.)
7. Cliquer "Importer" → INSERT dans Supabase
8. Voir un **rapport d'import** : X importés, Y doublons ignorés, Z
   erreurs

## Contexte CSV réel Blaïse Cendrars

Le CSV (export Pronote/STSWeb 2025-2026) contient **1158 élèves** mais
seulement **398 en lycée pro** (le reste = filière générale/techno
dont on se moque).

**Format réel observé (séparateur `;`, encoding ISO-8859-1) :**

```
NOM;PRENOM;SEXE;NE(E) LE;MEF;DIV.;REG.;OPT1;OPT2;...;DIV. PREC.;Doublant
AMIAR;Lyna;F;04/06/2009;PREMIERE GENERALE SELO;1G1;DEMI-PENSIONNAIRE;...
```

**Colonnes utiles pour PFMP Pilot :**
- `NOM` → `last_name`
- `PRENOM` → `first_name`
- `SEXE` → optionnel (M/F)
- `NE(E) LE` → date de naissance (format `dd/mm/yyyy`)
- `MEF` → libellé formation (= "BAC PRO MELEC", "CAP AGENT QUALITE
  ENVIRONNEMENT", etc.)
- `DIV.` → nom de classe (= "1MELEC1", "TPCEPC", "AQE1", "ETL2", etc.)

**Colonnes à ignorer :** `OPT1`-`OPT12`, `REG.`, `DIV. PREC.`, etc.

**Filtres à proposer à l'utilisateur :**
- "Importer uniquement certaines classes" (cases à cocher par classe
  détectée)
- Détection automatique : ne suggère que les classes pro (filtrage par
  préfixe `MELEC`, `PCEPC`, `AQE`, `ETL`, `2PRO`, etc.) **mais
  l'utilisateur peut toujours forcer**

## Contraintes non-négociables

### 🛡️ RGPD strict

- Le CSV contient des **données nominatives mineurs réels**
- Upload **côté serveur** ou **traité en mémoire client** (pas de
  storage permanent du fichier source sauf si demandé)
- Audit log **obligatoire** à chaque import (`action: 'import_students'`,
  metadata `{nb_imported, nb_errors, nb_duplicates}`)

### 🛡️ Encoding et séparateur

- **Auto-détection** : ISO-8859-1, UTF-8, UTF-8 BOM
- **Auto-détection séparateur** : `;`, `,`, tab
- Si auto-détection échoue, demander à l'utilisateur

### 🛡️ Dédoublonnage

Critère de doublon : `last_name` + `first_name` + `class_name` (à ce
stade, on n'a pas la date de naissance dans `students` schema).

Comportement par défaut : **skip** les doublons. Option "remplacer"
disponible mais cochée OFF par défaut.

### 🛡️ Mode démo

En mode démo, l'import simule (mock) la réussite mais n'écrit nulle
part. Affiche le rapport simulé.

### 🛡️ RLS

- Seul un user `admin` ou `ddfpt` peut importer
- L'`establishment_id` est forcé à celui du tenant courant (pas de
  cross-tenant)
