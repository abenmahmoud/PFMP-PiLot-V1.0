# P1.3 - Import SIECLE classes + eleves

> Brief Codex - sprint productivite etablissement
> Date : 11 mai 2026
> Depend de : P1.2.1 / P1.2.2 mergés

## Objectif fonctionnel

Un admin/DDFPT importe un fichier SIECLE et l'application cree automatiquement les classes et les eleves du tenant.

Ce sprint remplace l'ancien axe "Pronote d'abord". Pronote reste une compatibilite future, mais la cible MVP est SIECLE.

## Pourquoi

PFMP Pilot AI doit etre utilisable dans un vrai lycee pro sans saisir les eleves a la main et sans inviter les eleves par email.

Les eleves seront crees par l'etablissement, puis recevront plus tard des codes personnels par classe.

## Perimetre

### Route

Creer une route d'import, par exemple :

- `/imports/siecle`

ou, si la navigation existante le rend plus coherent :

- bouton "Importer SIECLE" depuis `/students`

### Service parsing

Creer :

- `src/services/imports/siecle.ts`

Le service doit :

- lire un CSV ou XLSX exporte depuis SIECLE ;
- detecter l'encodage si CSV ;
- detecter le separateur si CSV ;
- normaliser les noms de colonnes ;
- mapper les colonnes vers les champs internes ;
- detecter les classes/divisions ;
- detecter doublons ;
- preparer un plan d'import avant ecriture.

### Champs minimaux attendus

Champs cibles :

- `students.first_name`
- `students.last_name`
- `students.email` si disponible, mais optionnel
- `students.formation` depuis MEF/libelle formation si disponible
- `students.class_id`
- `classes.name`
- `classes.level`
- `classes.formation`
- `classes.school_year`

Le schema actuel `students` ne contient pas encore date de naissance, INE, sexe, responsables legaux. Ne pas ajouter de migration dans ce sprint sauf validation explicite.

### UI

Workflow en 4 etapes :

1. Upload du fichier SIECLE.
2. Preview des colonnes detectees + 10 premieres lignes.
3. Selection des divisions/classes a importer.
4. Confirmation : classes creees, eleves crees, eleves ignores/doublons.

### Ecriture Supabase

L'import doit :

- creer les classes manquantes ;
- reutiliser les classes existantes ;
- inserer les eleves ;
- ignorer les doublons par defaut ;
- creer un audit log `import.siecle.students`.

## Contraintes non-negociables

- RGPD strict : ne jamais commiter de fichier SIECLE reel.
- Le fichier source ne doit pas etre stocke en base ni Storage dans ce sprint.
- RLS naturelle : seul admin/DDFPT/superadmin avec tenant actif peut importer.
- Pas d'invitation email eleve.
- Pas de code eleve dans ce sprint : ce sera P1.4.
- Mode demo : simulation d'import sans ecriture.
- Typecheck et build verts.

## Critères d'acceptation

- [ ] Un fichier SIECLE test fictif peut etre importe.
- [ ] Les classes detectees s'affichent avant import.
- [ ] L'utilisateur peut choisir quelles classes importer.
- [ ] Les classes manquantes sont creees.
- [ ] Les eleves sont rattaches a leur classe.
- [ ] Les doublons sont detectes et ignores par defaut.
- [ ] Un rapport d'import est affiche.
- [ ] Un audit log est cree.
- [ ] Les eleves apparaissent ensuite dans `/students`.
- [ ] Aucun email eleve n'est requis.
