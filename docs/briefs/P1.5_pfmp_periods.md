## Salut Codex

Sprint qui ferme la phase P1. Sans périodes PFMP, aucune affectation
ne tient, aucune visite ne se rattache, aucun document ne se génère.

C'est le **squelette temporel** de toute l'année scolaire.

---

## Objectif fonctionnel

L'admin ou le DDFPT peut :

1. Aller sur `/pfmp-periods`
2. Voir la liste des périodes existantes (par année scolaire, statut)
3. Créer une nouvelle période :
   - Nom (ex : "P1 — Stage octobre 2025")
   - Année scolaire (ex : "2025-2026")
   - Date début / Date fin
   - Statut initial (`preparation`)
   - **Classes concernées** (multi-select)
4. Modifier une période existante (sauf si statut = `archived`)
5. Changer le statut : `preparation` → `in_progress` →
   `completed` → `archived`
6. Voir des **KPIs par période** :
   - Nombre élèves concernés
   - Nombre affectations effectuées
   - Nombre conventions générées
   - Nombre visites planifiées / réalisées

## Contraintes non-négociables

### 🛡️ Pas de chevauchement par classe

Une classe ne peut pas être dans **2 périodes en `in_progress`** en
même temps (sauf cas spécial type stage long fractionné — pas géré
maintenant).

→ À la création/modification, vérifier le chevauchement et bloquer
avec message clair : "La classe 1MELEC1 est déjà en stage du
17/03/2025 au 28/03/2025 (période P1 — Mars). Voulez-vous la déplacer ?"

### 🛡️ Statuts en cascade

- `preparation` → on peut tout modifier
- `in_progress` → date début, date fin, classes : verrouillés. Nom et
  notes : modifiables. Statut peut passer à `completed`.
- `completed` → seulement passage à `archived`. Sinon read-only.
- `archived` → totalement read-only

### 🛡️ Suppression

Suppression possible **uniquement si statut = preparation** ET aucun
élément lié (pas d'affectations, pas de visites). Sinon désactiver le
bouton avec tooltip explicatif.

### 🛡️ RLS et rôles

- `admin` ou `ddfpt` peut tout faire
- `principal` peut consulter
- `referent` ne voit pas cette page (RoleGuard)

### 🛡️ Mode démo

Garde fonctionnel. Mocke les KPIs.

## Périmètre

### Fichier 1 : `src/services/pfmpPeriods.ts`

```ts
export interface PfmpPeriodWithStats {
  id: string
  name: string
  schoolYear: string
  startDate: string
  endDate: string
  status: 'preparation' | 'in_progress' | 'completed' | 'archived'
  classIds: string[]
  classNames: string[]
  studentCount: number
  assignmentCount: number
  conventionCount: number
  visitsPlanned: number
  visitsCompleted: number
}

export async function fetchPfmpPeriods(filters?: {
  schoolYear?: string
  status?: string
}): Promise<PfmpPeriodWithStats[]>

export async function createPfmpPeriod(input: CreatePeriodInput): Promise<PfmpPeriod>
export async function updatePfmpPeriod(id: string, patch: PatchPeriodInput): Promise<PfmpPeriod>
export async function deletePfmpPeriod(id: string): Promise<void>
export async function changePeriodStatus(id: string, newStatus: string): Promise<PfmpPeriod>
export async function checkPeriodOverlap(input: {
  classIds: string[]
  startDate: string
  endDate: string
  excludePeriodId?: string  // pour update
}): Promise<{ hasOverlap: boolean; conflicts: ConflictInfo[] }>
```

### Fichier 2 : `src/routes/pfmp-periods.tsx`

Liste tableau avec :
- Filtre année scolaire

- Bouton "+ Nouvelle période"

Formulaire :

- Bouton "Créer"

→ Avant submit, appeler `checkPeriodOverlap` et bloquer si conflit.
