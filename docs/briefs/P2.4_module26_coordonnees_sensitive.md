## Salut Codex

Sprint **gouvernance produit**. Tu construis ici la **gestion des
informations légales du lycée** avec un workflow sécurisé pour les
champs sensibles (RNE, raison sociale, proviseur signataire,
assureur).

C'est délicat parce que :
- Il faut que l'admin lycée se sente **autonome** (pas appeler le
  support pour changer son téléphone)
- ET il faut **protéger** les champs juridiquement engageants
  (modifier le proviseur signataire d'une convention sans contrôle =
  risque légal)

Lis attentivement la **section "Module 26 — Sous-onglet Coordonnées"**
de `docs/VISION.md`.

---

## Objectif fonctionnel

L'admin (ou DDFPT) peut :

1. Aller sur `/settings/coordonnees`
2. Voir tous les champs de son établissement
3. Modifier directement les champs **non-sensibles** :
   - Téléphone
   - Email contact
   - Logo (upload image)
   - Couleur primaire
   - Adresse postale annexe
   - Familles de métiers acceptées (multi-select)
4. Pour les champs **sensibles** (RNE, raison sociale, proviseur,
   assureur, mentions légales conventions) :
   - Voir le badge `🔒 Champ sensible — validation superadmin requise`
   - Cliquer "Demander la modification"
   - Saisir la nouvelle valeur + un motif
   - Cliquer "Envoyer la demande"
   - La demande passe en statut `pending` et apparaît côté superadmin
5. Voir l'historique de ses demandes (en attente, approuvées,
   rejetées)

Le superadmin (BraveHeart) :

1. Voit dans `/superadmin/sensitive-validations` la file des demandes
   en attente
2. Pour chaque demande : voir l'avant/après, le motif, l'auteur
3. Approuver → modification appliquée + audit log + email
   confirmation à l'admin
4. Rejeter avec motif → demande clôturée + email à l'admin

(L'écran superadmin est dans P2.7. Ce sprint construit la partie
admin lycée et la table sous-jacente.)

## Contraintes non-négociables

### 🛡️ Liste figée des champs sensibles

Pas négociable. La liste est définie dans le code, pas dans la DB :

```typescript
export const SENSITIVE_ESTABLISHMENT_FIELDS = [
  'uai',           // RNE
  'rne',
  'raison_sociale',
  'proviseur_nom',
  'proviseur_signataire_legal',
  'assureur_nom',
  'assureur_numero',
  'mentions_legales_conventions',
] as const
```

Aucun moyen pour l'admin de contourner cette liste.

### 🛡️ Pas de modification directe des champs sensibles

L'admin **ne peut pas** faire un UPDATE direct sur ces champs. La RLS
le bloque côté DB en plus du UI. Triple protection :
1. UI désactivée
2. Service côté client refuse la modif
3. RLS Postgres bloque

### 🛡️ Audit log complet

Chaque demande de modification → entrée dans `sensitive_change_requests`
+ entrée dans `audit_logs` :
```
{
  action: 'sensitive_change_requested',
  target_type: 'establishment',
  target_id: <establishment_id>,
  metadata: {
    field: 'proviseur_nom',
    old_value: 'Mme VER EECKE',
    requested_value: 'M. DUPOND',
    reason: 'Changement de direction au 1er sept 2026',
    request_id: <new_request_id>
  }
}
```

### 🛡️ Notification superadmin en temps réel

Quand une demande est créée, le superadmin doit voir une **alerte**
dans son tableau de bord (compteur "X demandes en attente"). Pas
besoin d'email à ce stade — juste l'UI.

## Schema additionnel

### Migration `supabase/migrations/0007_sensitive_change_requests.sql`

```sql
create table if not exists sensitive_change_requests (
  id uuid primary key default gen_random_uuid(),
  establishment_id uuid not null references establishments(id) on delete cascade,
  requested_by uuid not null references profiles(id) on delete restrict,

  field_name text not null,
  old_value text,
  requested_value text not null,
  reason text not null,

  status text not null default 'pending'
    check (status in ('pending', 'approved', 'rejected', 'cancelled')),

  reviewed_by uuid references profiles(id) on delete set null,
  reviewed_at timestamptz,
  review_motive text,

  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists idx_scr_estab on sensitive_change_requests(establishment_id);
create index if not exists idx_scr_status on sensitive_change_requests(status);
create index if not exists idx_scr_requested_by on sensitive_change_requests(requested_by);

-- RLS
alter table sensitive_change_requests enable row level security;

-- L'admin lycée voit ses propres demandes
create policy "estab_admin_sees_own_requests"
  on sensitive_change_requests
  for select
  using (
    establishment_id = current_establishment_id()
    and current_role_in('admin', 'ddfpt')
  );

-- L'admin lycée peut créer des demandes pour son lycée
create policy "estab_admin_creates_own_requests"
  on sensitive_change_requests
  for insert
  with check (
    establishment_id = current_establishment_id()
    and current_role_in('admin', 'ddfpt')
    and requested_by = auth.uid()
    and status = 'pending'
  );

-- L'admin lycée peut annuler ses propres demandes en attente
create policy "estab_admin_cancels_own_requests"
  on sensitive_change_requests
  for update
  using (
    establishment_id = current_establishment_id()
    and current_role_in('admin', 'ddfpt')
    and requested_by = auth.uid()
    and status = 'pending'
  )
  with check (
    status = 'cancelled'
  );

-- Le superadmin voit toutes les demandes
create policy "superadmin_sees_all_requests"
  on sensitive_change_requests
  for all
  using (current_role_is_superadmin())
  with check (current_role_is_superadmin());

-- Trigger anti-cross-tenant
create trigger trg_prevent_scr_estab_change
  before update on sensitive_change_requests
  for each row execute function prevent_tenant_change();
```

### Augmentation `establishments`

```sql
alter table establishments
  add column if not exists assureur_nom text,
  add column if not exists assureur_numero text,
  add column if not exists proviseur_nom text,
  add column if not exists proviseur_signataire_legal text,
  add column if not exists raison_sociale text,
  add column if not exists rne text,
  add column if not exists mentions_legales_conventions text,
  add column if not exists logo_url text,
  add column if not exists primary_color text default '#1e40af',
  add column if not exists adresse_annexe text,
  add column if not exists telephone text,
  add column if not exists email_contact text,
  add column if not exists familles_metiers_acceptees text[] default array[]::text[];
```

exists` évite les conflits.)
