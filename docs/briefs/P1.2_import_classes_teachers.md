## Salut Codex

L'enjeu : qu'un admin puisse mettre toute sa structure pédagogique en
30 minutes maximum.

---

## Objectif fonctionnel

L'admin peut :

1. Aller sur `/import/classes` → upload CSV classes
2. Aller sur `/import/teachers` → upload CSV profs
3. Pour chaque : mapping intelligent + prévisualisation + import
4. Les profs importés ont un compte `auth.users` créé par invitation
   email (futur P1.4 ou option différée)

---

## Contexte CSV / format attendu

### Classes

L'admin peut soit :
- A) Importer un CSV simple : `name`, `level`, `formation`, `year`,
  `principal_email`
- B) **Plus probable** : laisser P1.1 auto-créer les classes lors de
  l'import élèves, puis venir les compléter ici

**Pour P1.2, gère le cas B en priorité** : page `/classes` avec
édition inline des classes auto-créées (ajouter level, formation,
prof principal email).

Et propose le cas A en bonus (CSV upload direct).

### Profs

Format CSV attendu (à proposer par défaut, modifiable) :

```
NOM;PRENOM;EMAIL;DISCIPLINE;CLASSES_PRINCIPAL;ROLES
DUPONT;Marc;m.dupont@blaisecendrars.fr;Génie électrique;1MELEC1,TMELEC2;referent,principal
LAMBERT;Élodie;e.lambert@blaisecendrars.fr;Lettres;;referent
```

Colonnes :
- `NOM`, `PRENOM`, `EMAIL` (obligatoire, sert de clé de dédoublonnage)
- `DISCIPLINE` (texte libre)
- `CLASSES_PRINCIPAL` (liste séparée par `,` des classes dont il est
  PP — optionnel)
- `ROLES` (liste séparée par `,` parmi : `admin`, `ddfpt`, `principal`,
  `referent`)

## Contraintes non-négociables

### 🛡️ Création de comptes auth — PRUDENT

**Décision produit :** lors de l'import profs, on crée les entrées
dans `teachers` (table métier) **MAIS PAS** automatiquement des
comptes `auth.users`. Pourquoi :

- Un compte auth implique un mot de passe → on ne peut pas le générer
  sans l'envoyer par email
- L'email d'invitation n'est pas encore branché (P1.4 ou +)
- Risque de spam / lockout si on crée 50 comptes d'un coup

**Comportement P1.2 :**
- INSERT dans `teachers` (avec email) → OK
- Pas d'INSERT dans `auth.users` ni de magic link à ce stade
- Statut prof : `imported_pending_invitation`
- Le DDFPT/admin pourra "Inviter" chaque prof manuellement plus tard
  (bouton dans la liste profs) — sprint P1.4 ou plus tard

### 🛡️ Dédoublonnage profs

Critère : `email` (lower-cased). Si email manquant → erreur sur la
ligne.

### 🛡️ Mapping classes ↔ profs principaux

Si la colonne `CLASSES_PRINCIPAL` contient `1MELEC1`, on cherche la
classe par son nom dans le tenant. Si trouvée → set
`classes.principal_id` au teacher importé. Si pas trouvée → warning
non-bloquant.

### 🛡️ Mode démo, RLS

Idem P1.1.
