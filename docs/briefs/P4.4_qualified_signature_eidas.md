## Salut Codex

Sprint **conformité juridique avancée**. La signature simple (P3.2)
suffit pour 95% des cas. Mais certains lycées (académies prudentes,
contentieux passés, exigences inspection) demandent la **signature
électronique qualifiée eIDAS** — celle qui a la même valeur qu'une
signature manuscrite **devant un juge sans contestation possible**.

C'est une option **Premium** vendue plus cher : intégration prestataire
externe agréé (Yousign, Universign, LexPersona, Docusign…).

⚠️ **Lis attentivement la section "Module 15 — Signatures qualifiées
eIDAS"** dans `docs/VISION.md`. Et **respecte strictement** : ne
JAMAIS écrire ou laisser entendre qu'une signature simple est
qualifiée. Mauvais wording = risque juridique majeur pour BraveHeart.

---

## Objectif fonctionnel

L'admin (ou DDFPT) peut, sur la page d'un placement :

1. Au lieu de "Envoyer le lien tuteur (signature simple)" cliquer
   sur "Demander signature qualifiée eIDAS"
2. Le système :
   - Vérifie que le tenant a l'option Premium activée (P4.5 Stripe)
   - Crée une **demande de signature qualifiée** via API
     prestataire (Yousign par exemple)
   - Le prestataire envoie un email au tuteur (et éventuellement au
     proviseur) avec son propre processus de signature qualifiée
   - Le tuteur signe via le prestataire (vérification d'identité par
     SMS, vidéo, OTP, etc.)
   - Le prestataire renvoie le PDF signé qualifié + le certificat de
     signature
3. PFMP Pilot stocke :
   - Le PDF signé qualifié dans `proof-files`
   - Le certificat de signature
   - Les metadata (date, méthode eIDAS, niveau, prestataire utilisé)
4. La fiche placement affiche : "✅ Convention signée qualifié eIDAS
   le DD/MM/YYYY par X (via Yousign)"

## Contraintes non-négociables

### ⚠️ Wording juridique strict

Sur l'UI, distinguer **clairement** :

- **Signature simple** : "Signature électronique simple par lien
  sécurisé. Article 1366 du Code civil. Pas eIDAS qualifiée."
- **Signature qualifiée** : "Signature électronique **qualifiée**
  eIDAS niveau 3 (le plus élevé). Vérification d'identité par
  prestataire agréé. Valeur juridique équivalente à une signature
  manuscrite."

Aucun mélange possible. Le bouton "Signer qualifié" ne s'affiche
**que** si le tenant a l'option Premium.

### 🛡️ Coût par signature

Chaque signature qualifiée coûte ~€2-5 chez les prestataires (varie
selon volume, contrat, niveau).

→ Track le coût par tenant pour facturation Premium :
- Ajouter colonne `qualified_signatures_count` dans
  `establishments`
- Compteur incrémenté à chaque signature réussie
- Permet à BraveHeart de facturer en fin de mois

### 🛡️ Choix du prestataire flexible

Le code doit permettre de **switcher** entre prestataires sans tout
réécrire. Architecture en **adapter pattern** :

```typescript
interface QualifiedSignatureProvider {
  name: 'yousign' | 'universign' | 'docusign' | 'lexpersona'

  createSignatureRequest(input: {
    documentBuffer: ArrayBuffer
    signers: Signer[]
    callbackUrl: string
  }): Promise<{ providerRequestId: string; signingUrl: string }>

  fetchStatus(providerRequestId: string): Promise<SignatureStatus>

  downloadSignedDocument(providerRequestId: string): Promise<{
    signedPdf: ArrayBuffer
    certificate: ArrayBuffer
    metadata: SignatureMetadata
  }>

  handleWebhook(payload: unknown): Promise<WebhookEvent | null>
}
```

Implémentation initiale : **Yousign** (français, RGPD-friendly,
documentation propre, pricing OK).

### 🛡️ Webhook sécurisé

Les prestataires notifient via webhook quand la signature est
finalisée. Endpoint à exposer + vérifier la signature HMAC du payload
pour éviter le spoofing.

### 🛡️ Pas de fallback dégradé

Si Yousign est down, on **ne dégrade pas** vers signature simple
silencieusement. Erreur claire à l'utilisateur : "Service de signature
qualifiée temporairement indisponible. Réessayez plus tard ou utilisez
la signature simple si suffisante."

### 🛡️ Stockage du certificat

Le **certificat de signature** est aussi important que le PDF signé.
Stocker les deux. Inclure dans le dossier de preuve (P3.3).

## Schema additionnel

### Migration `supabase/migrations/0015_qualified_signatures.sql`

```sql
create table if not exists qualified_signatures (
  id uuid primary key default gen_random_uuid(),
  establishment_id uuid not null references establishments(id) on delete cascade,
  placement_id uuid not null references placements(id) on delete cascade,
  document_id uuid references documents(id) on delete set null,

  -- Prestataire
  provider text not null check (provider in ('yousign', 'universign', 'docusign', 'lexpersona')),
  provider_request_id text not null,
  signing_url text,

  -- Statut
  status text not null default 'created'
    check (status in ('created', 'pending_signature', 'signed', 'rejected', 'expired', 'cancelled')),

  -- Signataires
  signers_json jsonb not null,
  -- [{name, email, role: 'tutor'|'proviseur'|'student', signed_at?, ...}]

  -- Résultat
  signed_pdf_path text,
  certificate_path text,
  signed_at timestamptz,
  signature_level text,  -- 'qualified_eidas_level_3', etc.

  -- Coût
  cost_eur numeric(10, 2),
  cost_eur_billed boolean default false,

  -- Méta
  callback_url text not null,
  expires_at timestamptz not null,
  created_by uuid references profiles(id) on delete set null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists idx_qsig_estab on qualified_signatures(establishment_id);
create index if not exists idx_qsig_placement on qualified_signatures(placement_id);
create index if not exists idx_qsig_status on qualified_signatures(status);
create index if not exists idx_qsig_provider_req on qualified_signatures(provider_request_id);

alter table qualified_signatures enable row level security;
create policy "qsig_tenant_isolation" on qualified_signatures
  for all
  using (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  )
  with check (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  );

create trigger trg_prevent_qsig_estab_change
  before update on qualified_signatures
  for each row execute function prevent_tenant_change();

-- Augmentation establishments
alter table establishments
  add column if not exists qualified_signatures_enabled boolean default false,
  add column if not exists qualified_signatures_count integer default 0;
```

## Périmètre

### Fichier 1 : Migration `0015_qualified_signatures.sql`

### Fichier 2 : Lib adapter `src/lib/qualifiedSignature/`

```
src/lib/qualifiedSignature/
├── index.ts              # exporte le provider courant
├── types.ts              # interfaces communes
├── yousign.ts            # implémentation Yousign
├── universign.ts         # stub pour futur (à implémenter quand demandé)
└── factory.ts            # createProvider(name) -> Provider
```

`yousign.ts` implémente l'interface en utilisant l'API Yousign v3 :
- `POST /v3/signature_requests`
- `POST /v3/documents`
- `POST /v3/signers`
- `POST /v3/signature_requests/{id}/activate`
- `GET /v3/signature_requests/{id}`
- `GET /v3/signature_requests/{id}/documents/{docId}/download`

API key stockée dans Supabase secrets : `YOUSIGN_API_KEY`.

  initie le processus

### Fichier 5 : UI

Sur la page placement, à côté du bouton "Envoyer signature simple",
un bouton "Signature qualifiée eIDAS" (visible uniquement si
`establishments.qualified_signatures_enabled = true`) :

```tsx
{tenant.qualified_signatures_enabled ? (
```

du process en cours.
