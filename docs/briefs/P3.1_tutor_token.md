## Salut Codex

Sprint qui ouvre **le workflow tuteur**. Aujourd'hui, le tuteur de
l'entreprise n'a aucun accès au système — il reçoit un PDF par email
ou en main propre, le signe, le scanne, le renvoie.

Avec ce sprint, le tuteur reçoit un **lien sécurisé** par email qui lui
permet d'accéder à une fiche stage limitée et de prochainement signer
en ligne (P3.2).

Lis attentivement la **section Module 15 — Signatures** de
`docs/VISION.md` avant de commencer.

---

## Objectif fonctionnel

### Côté prof référent / admin

1. Sur la fiche d'un placement, bouton "Envoyer au tuteur"
2. Le système :
   - Vérifie que l'email tuteur existe
   - Vérifie qu'une convention PDF a été générée (P2.6)
   - Génère un token unique sécurisé
   - Crée une ligne dans `tutor_access_tokens`
   - Envoie un email au tuteur (mocké via `email_queue` pour l'instant)
   avec le lien `https://<tenant>.pfmp-pilot.fr/sign/{token}`
3. Confirmation UI : "Lien envoyé à tuteur@entreprise.com"

### Côté tuteur

1. Ouvre l'email, clique le lien
2. Arrive sur `/sign/:token` (page publique, pas de login)
3. Vérification serveur : token valide, pas expiré, pas déjà utilisé
4. Voit une **fiche stage limitée** (lecture seule pour ce sprint) :
   - Nom de l'élève (sans données privées superflues)
   - Classe et formation
   - Dates du stage
   - Activités prévues
   - Coordonnées du lycée (qui contacter)
   - Coordonnées du professeur référent
   - Lien de téléchargement de la convention PDF (signed URL)
   - **Pas encore de bouton "Signer"** (vient en P3.2)
5. Bouton "J'accuse réception" qui marque le token comme `viewed_at`

## Contraintes non-négociables

### 🛡️ Sécurité du token

- Token = chaîne aléatoire **256 bits** (32 bytes encodés en base64url
  ≈ 43 caractères)
- Stocké en DB **hashé** (SHA-256), jamais en clair
- Le lien email contient le token clair, le serveur le hash pour
  comparer
- **Jamais loggué** (ni dans audit_logs, ni dans console)

### 🛡️ Expiration

- Par défaut : **30 jours** après création
- Configurable par le service (paramètre)
- Au-delà : token rejeté avec page d'erreur claire ("Lien expiré,
  contactez le professeur référent")

### 🛡️ Single-use ou multi-use ?

**Décision produit :** multi-use pour la consultation (le tuteur peut
revenir voir la fiche), single-use pour la signature (P3.2).

→ Pour P3.1 : `consultation_count`, `last_viewed_at`. Pas de blocage
sur N consultations.

### 🛡️ Pas de session, pas de cookie

La page `/sign/:token` est **stateless**. Pas d'auth Supabase. Le
token EST l'authentification. Pas de cookie.

### 🛡️ Données limitées

Le tuteur **ne doit pas** voir :
- Les autres élèves de la classe
- Les autres conventions
- Les notes internes du prof
- Les visites de stage
- Les autres tuteurs/entreprises
- Le numéro de téléphone personnel de l'élève (sauf cas explicitement
  validé par admin)

Il voit **uniquement** : l'élève dont il est tuteur, sur cette période,
avec des infos opérationnelles.

### 🛡️ Audit log

Chaque consultation tuteur :
```
{
  action: 'tutor_token_viewed',
  target_type: 'tutor_access_token',
  target_id: <token_id>,
  metadata: {
    placement_id, student_id, ip, user_agent, view_count
  }
}
```

(IP et UA collectés pour traçabilité juridique — RGPD-compatible si
mentionné dans politique confidentialité.)

### 🛡️ Pas de spam

Limite : 1 token actif par couple (tuteur, placement). Si on tente
d'en créer un nouveau alors qu'un est actif, on **réutilise** le
précédent (ou on **invalide** le précédent et on en crée un nouveau —
choix admin).

## Schema additionnel

### Migration `supabase/migrations/0010_tutor_access_tokens.sql`

```sql
create table if not exists tutor_access_tokens (
  id uuid primary key default gen_random_uuid(),
  establishment_id uuid not null references establishments(id) on delete cascade,
  placement_id uuid not null references placements(id) on delete cascade,
  tutor_id uuid not null references tutors(id) on delete cascade,

  -- Token sécurisé (HASHÉ, jamais en clair)
  token_hash text not null,
  token_prefix text not null,  -- 8 premiers caractères pour debug
  -- Le token clair n'est jamais stocké, seulement envoyé une fois par email

  -- Validité
  expires_at timestamptz not null,
  revoked_at timestamptz,

  -- Tracking
  consultation_count integer not null default 0,
  first_viewed_at timestamptz,
  last_viewed_at timestamptz,
  last_viewed_ip text,
  last_viewed_user_agent text,

  -- Lien vers signature future (P3.2)
  signed_at timestamptz,
  signature_method text,  -- 'simple' | 'qualified' (futur eIDAS)

  created_by uuid references profiles(id) on delete set null,
  created_at timestamptz not null default now(),

  unique (placement_id, tutor_id, revoked_at)
  -- Permet plusieurs tokens si le précédent est revoked (revoked_at != null)
);

create index if not exists idx_tat_token_hash on tutor_access_tokens(token_hash);
create index if not exists idx_tat_placement on tutor_access_tokens(placement_id);
create index if not exists idx_tat_estab on tutor_access_tokens(establishment_id);
create index if not exists idx_tat_expires on tutor_access_tokens(expires_at);

-- RLS
alter table tutor_access_tokens enable row level security;

-- Le tenant peut voir/créer ses propres tokens
create policy "tat_tenant_isolation"
  on tutor_access_tokens
  for all
  using (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  )
  with check (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  );

-- Edge Function avec service_role contournera la RLS pour la lecture par token
-- (le tuteur n'est pas authentifié, donc impossible via RLS pure)

create trigger trg_prevent_tat_estab_change
  before update on tutor_access_tokens
  for each row execute function prevent_tenant_change();
```

**Hash function :**

```typescript
async function hashToken(tokenClear: string): Promise<string> {
  const encoder = new TextEncoder()
  const data = encoder.encode(tokenClear)
  const hashBuffer = await crypto.subtle.digest('SHA-256', data)
  return Array.from(new Uint8Array(hashBuffer))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('')
}
```

### Fichier 3 : Edge Function `supabase/functions/resolve-tutor-token/index.ts`

Edge Function qui :
1. Reçoit le token clair en paramètre URL ou body
2. Hash le token
3. Cherche en DB via service_role
4. Vérifie : pas expiré, pas révoqué
5. Retourne les données limitées de la fiche stage
6. Incrémente `consultation_count`, met à jour `last_viewed_at`,
   `last_viewed_ip`, `last_viewed_user_agent`
7. Insère audit log

```typescript
serve(async (req) => {
  const { token } = await req.json()
  if (!token || typeof token !== 'string') {
    return new Response('Bad request', { status: 400 })
  }

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!,
  )

  const tokenHash = await hashToken(token)

  // Cherche le token
  const { data: tokenRow } = await supabase
    .from('tutor_access_tokens')
    .select(`
      *,
      placement:placements(*,
        student:students(first_name, last_name, class:classes(name, formation_referential:formation_referential(formation_label))),
        company:companies(name, address, city),
        tutor:tutors(first_name, last_name, email, phone)
      ),
      establishment:establishments(name, telephone, email_contact, proviseur_nom)
    `)
    .eq('token_hash', tokenHash)
    .is('revoked_at', null)
    .single()

  if (!tokenRow) {
    return new Response(JSON.stringify({ error: 'invalid_token' }), {
      status: 404,
      headers: { 'content-type': 'application/json' }
    })
  }

  if (new Date(tokenRow.expires_at) < new Date()) {
    return new Response(JSON.stringify({ error: 'expired_token' }), {
      status: 410,  // Gone
      headers: { 'content-type': 'application/json' }
    })
  }

  // Update tracking
  const ip = req.headers.get('x-forwarded-for') ?? 'unknown'
  const ua = req.headers.get('user-agent') ?? 'unknown'

  await supabase
    .from('tutor_access_tokens')
    .update({
      consultation_count: tokenRow.consultation_count + 1,
      first_viewed_at: tokenRow.first_viewed_at ?? new Date().toISOString(),
      last_viewed_at: new Date().toISOString(),
      last_viewed_ip: ip,
      last_viewed_user_agent: ua,
    })
    .eq('id', tokenRow.id)

  // Audit log
  await supabase.from('audit_logs').insert({
    establishment_id: tokenRow.establishment_id,
    action: 'tutor_token_viewed',
    target_type: 'tutor_access_token',
    target_id: tokenRow.id,
    metadata: {
      placement_id: tokenRow.placement_id,
      student_id: tokenRow.placement.student.id,
      ip,
      user_agent: ua,
      consultation_count: tokenRow.consultation_count + 1,
    }
  })

  // Retour données limitées
  return Response.json({
    student: {
      first_name: tokenRow.placement.student.first_name,
      last_name: tokenRow.placement.student.last_name,
      class_name: tokenRow.placement.student.class.name,
      formation: tokenRow.placement.student.class.formation_referential?.formation_label,
    },
    placement: {
      start_date: tokenRow.placement.start_date,
      end_date: tokenRow.placement.end_date,
      activities: tokenRow.placement.activities,
    },
    tutor: {
      first_name: tokenRow.placement.tutor.first_name,
      last_name: tokenRow.placement.tutor.last_name,
      email: tokenRow.placement.tutor.email,
    },
    establishment: {
      name: tokenRow.establishment.name,
      telephone: tokenRow.establishment.telephone,
      email_contact: tokenRow.establishment.email_contact,
      proviseur_nom: tokenRow.establishment.proviseur_nom,
    },
    convention_pdf_url: '...',  // signed URL générée à la volée si convention dispo
    expires_at: tokenRow.expires_at,
    can_sign: false,  // false en P3.1, true en P3.2
  })
})
```
