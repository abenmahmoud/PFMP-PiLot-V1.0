---

## Salut Codex

Suite logique de P0.7 : une visite créée en brouillon doit pouvoir être
relue et **validée**. Une fois validée, elle est officiellement
enregistrée dans le dossier de l'élève et apparaît dans le dashboard
DDFPT.

---

## Objectif fonctionnel

Un prof peut :

1. Aller sur `/visits/$id` d'une de ses visites en brouillon
2. Relire le contenu, modifier si besoin
3. Cliquer "Valider le compte rendu"
4. Le statut passe `brouillon` → `validated`
5. Audit log enregistré
6. La visite apparaît dans le dashboard DDFPT (côté DDFPT du tenant)

Un DDFPT peut :

1. Voir la liste des visites de son tenant
2. Voir le détail d'une visite (lecture seule s'il n'est pas auteur)
3. Marquer "lue" ou "à corriger"

---

## Contraintes

- RLS : seul l'auteur peut valider/modifier sa propre visite
- DDFPT peut lire toutes les visites de son tenant (RLS via role)
- Audit log à chaque transition statut
