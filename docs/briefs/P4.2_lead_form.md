---

## Salut Codex

Sprint **capture commerciale**. Le formulaire `/contact` est ce qui
transforme un visiteur intéressé en prospect concret pour
BraveHeart.

C'est court (2-3h de code) mais critique : un formulaire mal foutu
fait perdre des leads.

---

## Objectif fonctionnel

Un visiteur de la landing peut :

1. Cliquer "Demander une démo" ou "Demander un devis"
2. Arriver sur `/contact`
3. Remplir un formulaire structuré :
   - Nom du lycée *
   - UAI / RNE * (8-9 caractères, format `0XXXXXXXY`)
   - Ville *
   - Académie * (dropdown)
   - Nom du contact *
   - Fonction * (Proviseur, DDFPT, autre)
   - Email contact *
   - Téléphone (optionnel)
   - Métiers enseignés (multi-select : Bac Pro, CAP, BTS, formations
     spécifiques)
   - Nombre d'élèves concernés (estimation)
   - Plan envisagé : Standard / Premium IA / "À discuter"
   - Add-on Mise en place : oui / non
   - Date souhaitée d'ouverture (mois)
   - Message libre (optionnel)
4. Cliquer "Envoyer la demande"
5. Le formulaire :
   - Valide les champs côté client
   - POST vers une API (Edge Function ou route Vercel API)
   - Envoie un email à BraveHeart (`abenmahmoud@essuf.fr` ou similaire)
     avec toutes les infos
   - Crée une entrée dans `lead_requests` (nouvelle table) pour
     historique
   - Affiche un message de confirmation : "Votre demande est bien
     reçue. BraveHeart vous recontacte sous 24h."

## Contraintes non-négociables

### 🛡️ Anti-spam

- Honeypot (champ caché que les bots remplissent)
- Rate limiting (max 3 soumissions par IP / heure)
- Pas de captcha invasif type reCAPTCHA Google (RGPD-flou)
- Optionnel : challenge maths simple si suspicion

### 🛡️ Validation côté serveur

Tous les champs * sont vérifiés côté serveur. UAI doit matcher
regex `^0\d{6}[A-Z]$` (RNE français).

### 🛡️ RGPD

- Mention obligatoire sous le formulaire : "Vos données sont
  utilisées uniquement pour traiter votre demande. Voir notre
  politique de confidentialité."
- Case à cocher (pas pré-cochée) : "J'accepte que mes données soient
  utilisées pour me recontacter"
- Pas d'envoi à un tracker

### 🛡️ Email à BraveHeart

Format clair :

```
Sujet : 🎯 Nouvelle demande PFMP Pilot - <Nom Lycée>

Lycée : Lycée X (UAI 0XXXXXXX)
Ville : ... | Académie : ...

Contact :
  Nom : ...
  Fonction : ...
  Email : ...
  Téléphone : ...

Besoin :
  Métiers : Bac Pro MELEC, CAP AQE, ...
  Élèves estimés : ~150
  Plan : Standard
  Mise en place : oui
  Ouverture souhaitée : septembre 2026

Message :
  ...

---
Voir dans le backoffice : https://app.pfmp-pilot.fr/superadmin/leads
```

## Schema additionnel

### Migration `supabase/migrations/0013_lead_requests.sql`

```sql
create table if not exists lead_requests (
  id uuid primary key default gen_random_uuid(),

  -- Lycée
  lycee_name text not null,
  lycee_uai text not null,
  lycee_ville text not null,
  lycee_academie text not null,

  -- Contact
  contact_name text not null,
  contact_function text not null,
  contact_email text not null,
  contact_phone text,

  -- Besoin
  metiers_enseignes text[] not null default array[]::text[],
  nb_eleves_estime integer,
  plan_souhaite text check (plan_souhaite in ('standard', 'premium_ia', 'a_discuter')),
  miseenplace boolean default false,
  date_ouverture_souhaitee text,  -- format "YYYY-MM"
  message text,

  -- Tracking
  source_url text,
  source_referrer text,
  ip text,
  user_agent text,

  -- Statut côté Essuf
  status text not null default 'new'
    check (status in ('new', 'contacted', 'in_discussion', 'converted', 'rejected', 'lost')),
  notes_internes text,
  assigned_to uuid references profiles(id) on delete set null,

  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists idx_lead_requests_status on lead_requests(status);
create index if not exists idx_lead_requests_created on lead_requests(created_at desc);

-- RLS : superadmin only
alter table lead_requests enable row level security;
create policy "lead_requests_superadmin" on lead_requests
  for all using (current_role_is_superadmin()) with check (current_role_is_superadmin());

-- L'insertion publique se fait via Edge Function avec service_role
```
