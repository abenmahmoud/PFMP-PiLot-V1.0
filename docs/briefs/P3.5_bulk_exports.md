> **Brief Codex** — sprint final P3

## Salut Codex

Sprint qui ferme P3. Tu étends le **dossier de preuve par élève**
(P3.3) à des **exports en lot** (par classe, période, année scolaire).

C'est ce que le DDFPT utilisera **en fin d'année** pour archiver
toutes les PFMP de l'établissement, les transmettre à l'académie, ou
juste pour la sauvegarde annuelle obligatoire.

---

## Objectif fonctionnel

Le DDFPT (ou admin) peut :

1. Aller sur `/exports`
2. Voir 3 modes d'export :
   - **Par élève** (renvoie vers P3.3)
   - **Par classe + période** (nouveau)
   - **Par année scolaire complète** (nouveau)
3. Sélectionner une classe + période → bouton "Exporter ZIP groupé"
4. Le système :
   - Itère sur tous les élèves de la classe pour cette période
   - Pour chaque élève : génère son dossier individuel (réutilise P3.3)
   - Combine tous les dossiers dans un **ZIP global**
   - Ajoute un `index.json` global qui liste tous les élèves
5. Téléchargement ZIP via signed URL (peut être gros — 50-200 MB pour
   une classe entière)

Et un mode **archive année** :

6. Sélectionner une année scolaire → tous les élèves de toutes les
   classes de cette année
7. ZIP encore plus gros (ou fragmenté en plusieurs ZIP de 100 MB max)

---

## Contraintes non-négociables

### 🛡️ Performance — async + queue

Un ZIP de 200 MB ne se génère pas en 5s. **Stratégie** :

1. Le clic "Exporter" crée un **job** en DB (table `export_jobs`)
2. L'utilisateur reçoit un message "Export en cours, vous serez
   notifié dès qu'il sera prêt (durée estimée : X minutes)"
3. Edge Function ou worker traite le job en arrière-plan
4. Quand fini : `export_jobs.status = 'ready'` + `download_url` (signed
   URL valide 24h)
5. L'utilisateur reçoit une notification UI (et email à terme)
6. Il peut télécharger depuis `/exports` (liste des jobs récents)

### 🛡️ Storage temporaire

Les ZIP générés vont dans `proof-files` bucket avec sous-dossier
`exports-bulk/`. Auto-cleanup après 7 jours pour ne pas remplir le
storage.

### 🛡️ Permissions

- DDFPT / admin : exports tous niveaux
- Prof principal : export classe (uniquement ses classes)
- Référent : pas d'export bulk (juste P3.3 individuel)

### 🛡️ Anonymisation optionnelle

Option : "Anonymiser les noms d'élèves" (utile pour partage externe
type formation, audit académie sans nominatif).

Si coché : remplace nom/prénom par "Élève_001, Élève_002..." dans le
manifest et dans les noms de fichiers PDF (pas dans les PDF
eux-mêmes — ils gardent les vrais noms car ce sont des documents
légaux signés).

## Schema additionnel

### Migration `supabase/migrations/0012_export_jobs.sql`

```sql
create table if not exists export_jobs (
  id uuid primary key default gen_random_uuid(),
  establishment_id uuid not null references establishments(id) on delete cascade,
  requested_by uuid not null references profiles(id) on delete restrict,

  job_type text not null check (job_type in ('student', 'class_period', 'school_year')),
  scope_json jsonb not null,
  -- ex: { class_id, period_id, school_year, anonymize, ... }

  status text not null default 'pending'
    check (status in ('pending', 'running', 'ready', 'failed', 'expired')),

  total_students integer,
  processed_students integer default 0,
  download_url text,
  download_expires_at timestamptz,
  storage_path text,
  size_bytes bigint,
  error_message text,

  started_at timestamptz,
  completed_at timestamptz,
  created_at timestamptz not null default now()
);

create index if not exists idx_export_jobs_estab on export_jobs(establishment_id);
create index if not exists idx_export_jobs_status on export_jobs(status);
create index if not exists idx_export_jobs_user on export_jobs(requested_by);

alter table export_jobs enable row level security;
create policy "export_jobs_isolation" on export_jobs
  for all
  using (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  )
  with check (
    establishment_id = current_establishment_id()
    or current_role_is_superadmin()
  );

create trigger trg_prevent_export_jobs_estab_change
  before update on export_jobs
  for each row execute function prevent_tenant_change();
```

## Périmètre

### Fichier 1 : Migration `0012_export_jobs.sql`

Voir SQL ci-dessus.

### Fichier 2 : Edge Function `supabase/functions/process-export-job/index.ts`

Worker qui :
1. Cherche un job `pending`, le marque `running`
2. Itère sur les élèves de la scope
3. Pour chaque élève : appelle la logique P3.3 (dossier de preuve
   individuel)
4. Concatène tous les ZIP dans un méga-ZIP
5. Upload dans storage
6. Met à jour `export_jobs` : `status='ready'`, `download_url`, etc.

**Trigger** : invocation manuelle via Edge Function ou cron (à
définir).
