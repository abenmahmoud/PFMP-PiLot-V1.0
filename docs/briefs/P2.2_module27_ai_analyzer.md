## Salut Codex

**Sprint le plus important de la phase commerciale.**

Ce module justifie 100% du **service "Mise en place" payant**. Sans
lui, configurer un nouveau lycée demanderait 2-3 jours de saisie
manuelle. Avec lui, c'est une demi-journée d'IA + validation humaine.

C'est aussi un **vrai défi technique** : faire causer Claude à un
.docx structuré pour en extraire des grilles de compétences propres,
avec un humain dans la boucle pour valider.

Lis attentivement la **section Module 27** de `docs/VISION.md` avant
de commencer.

## Objectif fonctionnel

Le superadmin (BraveHeart) peut :

1. Aller sur `/superadmin/referential-analyzer`
2. Sélectionner un référentiel cible (ex : `BAC_PRO_MELEC` année 1)
   ou créer un nouveau
3. Uploader 1 à 4 fichiers source du lycée :
   - Convention vierge (.docx ou .pdf)
   - Livret PFMP (.docx) — un par formation
   - Attestation type (.docx)
   - Référentiel compétences si disponible (.pdf, ex : doc Eduscol)
4. Cliquer "Analyser avec l'IA"
5. **L'IA analyse et propose un mapping structuré** :
   - Champs dynamiques détectés dans la convention
   - Attitudes professionnelles AP1-APn détectées dans le livret
   - Compétences C1-Cn détectées dans le livret/référentiel
   - Niveaux d'évaluation détectés (échelle 4 niveaux par exemple)
6. Le superadmin **valide / ajuste / corrige** chaque détection :
   - Modifie un libellé d'attitude
   - Ajoute une compétence manquante
   - Supprime un faux positif
7. Cliquer "Sauvegarder le référentiel"
8. Le `formation_referential` est mis à jour avec les `attitudes_grid`
   et `competences_grid` pleines
9. Le `establishment_formation_setup` est créé/mis à jour avec les
   chemins des templates uploadés (Supabase Storage)

## Contraintes non-négociables

### 🛡️ Superadmin only

- Route `/superadmin/referential-analyzer` accessible **uniquement**
  aux users `role = 'superadmin'`
- Edge Function `analyze-referential` vérifie le rôle côté serveur
  (sécurité défensive : pas confiance dans le RoleGuard côté client)

### 🛡️ Pas de hallucination IA

- Le prompt IA doit être **strict** : "n'invente rien, retourne
  uniquement ce que tu trouves dans le document"
- Si l'IA n'est pas sûre d'une détection, elle doit le signaler
  (`confidence: low`)
- L'IA retourne du JSON structuré, pas du texte libre

### 🛡️ Validation humaine obligatoire

- Aucun référentiel n'est sauvegardé sans validation explicite par le
  superadmin (clic sur bouton "Sauvegarder")
- L'utilisateur doit voir et pouvoir éditer **chaque champ détecté**
  avant sauvegarde

### 🛡️ Logs IA dans `ai_interactions`

Chaque appel à l'IA est logué :
```
{
  user_id: <superadmin>,
  assistant_type: 'referential_analyzer',
  request: { files_count, file_types, target_referential_id },
  response: { attitudes_count, competences_count, confidence, model_used },
  tokens_used: ...,
  duration_ms: ...,
  created_at: ...
}
```

### 🛡️ Stockage des sources dans Storage

Les .docx uploadés sont stockés dans le bucket `referentials-source`
(privé) avec un chemin structuré :
`{establishment_id}/{formation_code}_y{year_level}/{document_type}_{timestamp}.docx`

L'admin/DDFPT du lycée pourra les retélécharger via le Module 26 (P2.5)
mais ne peut pas les supprimer.

### 🛡️ Coût IA contrôlé

- Limite : 1 appel = max 4 fichiers, max 50 pages cumulées
- Modèle utilisé : Claude Sonnet (rapport coût/qualité optimal)
- Si le superadmin spamme : rate limit 10 appels/heure

### Format de réponse IA

```typescript
interface AnalyzerResponse {
  attitudes: {
    code: string          // "AP1"
    label: string         // "Intérêt porté au travail"
    confidence: 'high' | 'medium' | 'low'
    levels: {
      value: number       // 1 à 4
      label: string       // "Exécute sans intérêt"
    }[]
    sourceQuote?: string  // Phrase exacte du doc source pour audit
  }[]

  competences: {
    code: string          // "C1.1"
    label: string         // "Préparer une opération"
    confidence: 'high' | 'medium' | 'low'
    indicators?: string[]
    pfmpEvaluable: boolean
    sourceQuote?: string
  }[]

  conventionFields: {
    placeholder: string   // "{{eleve_nom}}" ou variant détecté
    inferredMeaning: string  // "Nom de famille de l'élève"
    confidence: 'high' | 'medium' | 'low'
  }[]

  globalConfidence: number  // 0-100
  warnings: string[]
  modelUsed: string
  tokensUsed: number
}
```

### Fichier 1 : Edge Function `supabase/functions/analyze-referential/index.ts`

```typescript
import { serve } from "std/http/server.ts"
import { createClient } from "@supabase/supabase-js"
import Anthropic from "@anthropic-ai/sdk"
import mammoth from "mammoth"

serve(async (req) => {
  // 1. Auth & rôle superadmin
  const authHeader = req.headers.get('Authorization')
  if (!authHeader) return new Response('Unauthorized', { status: 401 })

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!,
    { global: { headers: { Authorization: authHeader } } }
  )

  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return new Response('Unauthorized', { status: 401 })

  const { data: profile } = await supabase
    .from('profiles')
    .select('role')
    .eq('id', user.id)
    .single()
  if (profile?.role !== 'superadmin') {
    return new Response('Forbidden — superadmin only', { status: 403 })
  }

  // 2. Parse body
  const { storagePaths, targetReferentialId, formationCode, yearLevel } = await req.json()

  // 3. Download files from storage
  const documents: { name: string; type: string; text: string }[] = []
  for (const path of storagePaths) {
    const { data: blob } = await supabase.storage
      .from('referentials-source')
      .download(path)
    if (!blob) continue

    const buffer = await blob.arrayBuffer()
    const ext = path.split('.').pop()?.toLowerCase()

    let text = ''
    if (ext === 'docx') {
      const result = await mammoth.extractRawText({ arrayBuffer: buffer })
      text = result.value
    } else if (ext === 'pdf') {
      // utiliser pdf-parse ou similaire
      text = await extractPdfText(buffer)
    }
    documents.push({ name: path, type: ext!, text })
  }

  // 4. Build prompt
  const prompt = buildAnalysisPrompt(documents, formationCode, yearLevel)

  // 5. Call Anthropic
  const anthropic = new Anthropic({
    apiKey: Deno.env.get('ANTHROPIC_API_KEY')!,
  })

  const startTs = Date.now()
  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-7',
    max_tokens: 4096,
    messages: [{ role: 'user', content: prompt }],
  })
  const durationMs = Date.now() - startTs

  // 6. Parse response
  const text = response.content[0].type === 'text' ? response.content[0].text : ''
  const parsed = extractJsonFromResponse(text)

  // 7. Log interaction
  await supabase.from('ai_interactions').insert({
    user_id: user.id,
    assistant_type: 'referential_analyzer',
    request: { storagePaths, formationCode, yearLevel },
    response: {
      attitudes_count: parsed.attitudes.length,
      competences_count: parsed.competences.length,
      conventionFields_count: parsed.conventionFields.length,
      globalConfidence: parsed.globalConfidence,
    },
    tokens_used: response.usage.input_tokens + response.usage.output_tokens,
    duration_ms: durationMs,
  })

  return Response.json(parsed)
})
```

### Fichier 2 : Le prompt (le cerveau du sprint)

```typescript
function buildAnalysisPrompt(
  documents: { name: string; type: string; text: string }[],
  formationCode: string,
  yearLevel: number
): string {
  return `Tu es un assistant expert en référentiels de formation professionnelle française.

Ta mission : analyser les documents officiels fournis pour la formation
${formationCode} année ${yearLevel} et en extraire :

1. Les **attitudes professionnelles** évaluées (AP1, AP2, AP3...) avec leurs niveaux
2. Les **compétences** professionnelles (C1.1, C1.2, ou autre nomenclature trouvée)
3. Les **champs dynamiques** détectés dans la convention de stage (champs vides à remplir)

CONSIGNES STRICTES :
- N'invente AUCUNE attitude, compétence ou champ qui n'apparaît pas dans les documents.
- Si tu détectes quelque chose mais que tu n'es pas sûr, marque-le \`confidence: 'low'\`.
- Cite le passage exact du document source pour chaque détection (\`sourceQuote\`).
- Pour les attitudes professionnelles : extrait les libellés des niveaux (ex : "Aucun intérêt", "Peu d'écoute", "Bonne participation", "Excellent").
- Pour les compétences : note si la compétence est explicitement marquée comme évaluable en PFMP.
- Pour les champs convention : repère les espaces vides, lignes "..............", placeholders {{ }} ou éléments à compléter manuellement.

Documents à analyser :

${documents.map((d, i) => `=== DOCUMENT ${i + 1} : ${d.name} (${d.type}) ===
${d.text.slice(0, 15000)}
${d.text.length > 15000 ? '\n[... document tronqué à 15000 caractères ...]' : ''}
`).join('\n\n')}

Réponds UNIQUEMENT par un JSON valide au format suivant (pas de markdown, pas de commentaire) :

{
  "attitudes": [
    {
      "code": "AP1",
      "label": "Intérêt porté au travail",
      "confidence": "high",
      "levels": [
        { "value": 1, "label": "Exécute sans intérêt" },
        { "value": 2, "label": "Peu d'écoute et de curiosité" },
        { "value": 3, "label": "S'implique" },
        { "value": 4, "label": "Excellent investissement" }
      ],
      "sourceQuote": "AP1 INTÉRÊT PORTÉ AU TRAVAIL"
    }
  ],
  "competences": [
    {
      "code": "C1.1",
      "label": "...",
      "confidence": "high",
      "indicators": ["..."],
      "pfmpEvaluable": true,
      "sourceQuote": "..."
    }
  ],
  "conventionFields": [
    {
      "placeholder": "Nom de l'élève",
      "inferredMeaning": "student.last_name",
      "confidence": "high"
    }
  ],
  "globalConfidence": 85,
  "warnings": [
    "Document 2 incomplet, possibilité de manquer des compétences."
  ]
}`
}
```

## Méthode

Plan dans `docs/sprints/P2.2_plan.md`. **Décris explicitement** :

1. **Choix du modèle Anthropic** : `claude-sonnet-4-7` recommandé pour

3. **Stratégie de retry** si la réponse JSON n'est pas valide (parser
   en mode tolérant ? re-prompt avec correction ?)
4. **UI État de chargement** pendant l'analyse (loader avec étapes
   "Extraction texte..." → "Analyse IA..." → "Mise en forme...")
5. **Comment tu testes** sans cramer du budget API
