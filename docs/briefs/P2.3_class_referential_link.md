## Salut Codex

Sprint **pivot** : c'est ici que la **règle d'or référentielle** prend
vie pour les classes existantes.

À la fin de P1.2, le lycée Blaïse Cendrars a 13 classes en DB
(`1MELEC1`, `TPCEPC`, `AQE1`, etc.) **sans** rattachement à un
référentiel. Après P2.2, les référentiels nationaux MELEC/PCEPC/AQE/ETL
ont leurs grilles AP/compétences pleines.

Ce sprint connecte les deux : chaque classe **sait** quel est son
référentiel. À partir de là, tout se déduit automatiquement.

---

## Objectif fonctionnel

L'admin (ou DDFPT) peut :

1. Aller sur `/classes`
2. Voir la liste des classes du lycée
3. Pour chaque classe sans référentiel : **suggestion automatique**
   basée sur le code classe :
   - `1MELEC1` → suggestion `BAC_PRO_MELEC` année 2 (1ère)
   - `2NDE_MELEC` → `BAC_PRO_MELEC` année 1
   - `TPCEPC` → `BAC_PRO_PCEPC` année 3 (terminale)
   - `AQE1` → `CAP_AQE` année 1
4. Valider la suggestion ou choisir un autre référentiel dans une
   liste déroulante
5. Sauvegarder en lot ou en individuel
6. Voir un badge sur chaque classe : "✅ Référentiel : Bac Pro MELEC
   année 1" ou "⚠️ Référentiel non défini"

À la fin, **les 13 classes Blaïse Cendrars** sont toutes rattachées à
leur référentiel.

---

## Contraintes non-négociables

### 🛡️ Suggestion automatique = aide, pas obligation

L'admin **valide toujours**. Aucun rattachement automatique sans
clic.

### 🛡️ Pas de seed national pour Blaïse Cendrars

Si `1MELEC1` est rattachée au référentiel national `BAC_PRO_MELEC`
année 2 directement (pas de version locale Blaïse), c'est OK.
L'overriding viendra plus tard si Blaïse a des spécificités (Module
26, P2.5).

### 🛡️ Audit log

Chaque rattachement → `audit_logs` :
```
{
  action: 'class_referential_assigned',
  target_type: 'class',
  target_id: <class_id>,
  metadata: { previous_referential_id, new_referential_id }
}
```

### 🛡️ Mode démo

Page `/classes` continue de marcher en mode démo (les classes démo
n'ont pas de référentiel, c'est OK).

## Heuristique de suggestion automatique

Le pattern : `<niveau><CODE_FORMATION><groupe>`

| Préfixe classe | Année level | Code formation déduit |
|---|---|---|
| `2` ou `2NDE` ou `SEC` ou `2PRO` | 1 (seconde) | à déduire de la suite |
| `1` (mais pas 1G ou 1STMG ou 1STL) | 2 (première) | à déduire de la suite |
| `T` ou `TERM` | 3 (terminale) | à déduire de la suite |
| `AQE` | (suffix `1` ou `2`) | `CAP_AQE` année 1 ou 2 |
| `ETL` | (suffix `1` ou `2`) | `CAP_ETL` année 1 ou 2 |

Token de formation à reconnaître :
- `MELEC` → `BAC_PRO_MELEC`
- `PCEPC` → `BAC_PRO_PCEPC`
- `AQE` → `CAP_AQE`
- `ETL` → `CAP_ETL`
- `2PRO` (sans formation précisée) → suggérer **manuellement** car
  c'est une seconde commune Bac Pro qui sera spécialisée plus tard

Exemples :
- `1MELEC1` → 1 = première, MELEC = Bac Pro MELEC → `BAC_PRO_MELEC`
  année 2
- `1MELEC2` → idem
- `TMELEC1` → T = terminale, MELEC → `BAC_PRO_MELEC` année 3
- `1PCEPC` → 1 = première, PCEPC → `BAC_PRO_PCEPC` année 2
- `TPCEPC` → terminale → `BAC_PRO_PCEPC` année 3
- `2PRO1`, `2PRO2`, `2PRO3`, `2PRO4` → seconde Bac Pro **commune** →
  pas de suggestion automatique, demander à l'admin (probablement
  MELEC ou PCEPC année 1 selon orientation finale)
- `AQE1` → CAP AQE année 1
- `ETL2` → CAP ETL année 2

  classId: string,
