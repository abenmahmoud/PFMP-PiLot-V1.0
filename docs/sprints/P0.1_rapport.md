## Decisions techniques prises

- Copie demo realisee par Claude, conformement a `docs/sprints/P0.1_answers.md`, parce que l'environnement Codex avait tronque la copie initiale.
- Pas de patch `src/lib/useCurrentUser.ts` dans P0.1 : decision Claude. Le hook garde son contrat `Profile`, et le durcissement prod strict est reporte a P0.9 ou a un sprint P0.10 dedie.
- Pas de modification `src/components/EmptyState.tsx` : son API existante `action?: ReactNode` est plus flexible que l'API suggeree dans le brief.
- Audit imports uniquement : aucun fichier metier important `@/data/demo` n'a ete migre pendant P0.1.

## Comportement de useCurrentUser apres ce sprint

- Mode demo (`VITE_DEMO_MODE=true`) : retourne le profil demo choisi via `localStorage` (`pfmp_demo_user`) ou le `CURRENT_USER_ID` demo.
- Mode prod avec session Supabase et profil charge : retourne le profil Supabase mappe vers le shape applicatif par le code existant.
- Mode prod sans profil/session : fallback actuel vers le profil demo. C'est volontairement documente comme dette, pas corrige dans P0.1 pour ne pas casser `Sidebar`, `Topbar`, `RoleGuard` et les pages qui attendent un `Profile` non-null.

## TODO Auth strict prod

Le comportement actuel reste dangereux en prod vide : si `auth.profile` est absent, l'app peut encore se comporter comme si l'utilisateur demo etait connecte. Le futur sprint devra :

- introduire un garde en amont (`RoleGuard`, layout ou route loader) qui redirige vers `/login` avant que les composants ne lisent un profil demo ;
- decider si `useCurrentUser()` reste non-null en demo mais expose un hook strict prod separe ;
- auditer au minimum `Sidebar.tsx`, `Topbar.tsx`, `RoleGuard.tsx`, `src/routes/index.tsx` et toutes les pages qui dereferencent `firstName`, `lastName` ou `role` directement.

## EmptyState

`src/components/EmptyState.tsx` existe deja et expose :

- `icon?: ReactNode`
- `title: string`
- `description?: string`
- `action?: ReactNode`
- `className?: string`

API validee. L'action en `ReactNode` permet un `Link`, un `Button`, ou une action composee, donc elle est plus flexible que le brief initial.
