# P1.1 — Tenant switcher superadmin

## Objectif

Permettre de tester le mode SaaS multi-etablissement sans attendre le DNS wildcard.

## Probleme traite

Le composant `SuperadminTenantSwitcher` existait deja mais le mode incarnation restait incomplet :

- l'etat auth ne remontait pas explicitement `active_establishment_id` ;
- la sidebar restait limitee aux vues superadmin ;
- `RoleGuard` bloquait le superadmin sur les pages tenant ;
- les lectures UI principales restaient globales pour un superadmin, car la RLS autorise volontairement `is_superadmin()`.

## Changements

- `src/lib/auth.ts`
  - ajout de `activeEstablishmentId` dans `AuthState` ;
  - lecture de `user_metadata.active_establishment_id` ;
  - helper `getActiveEstablishmentScope()` pour les services.

- `src/components/SuperadminTenantSwitcher.tsx`
  - utilise `auth.activeEstablishmentId` ;
  - rafraichit la session apres changement de tenant ;
  - ajoute un lien rapide vers `/dashboard` quand un tenant est actif ;
  - remonte une erreur via `title` si le switch echoue.

- `src/components/Sidebar.tsx`
  - si un superadmin a un tenant actif, la navigation etablissement devient visible.

- `src/components/RoleGuard.tsx`
  - autorise le superadmin a ouvrir les pages tenant uniquement quand un tenant actif est choisi.

- Services scopes par tenant actif :
  - dashboard
  - students
  - companies
  - classes
  - teachers
  - pfmp periods
  - documents
  - assignments
  - alerts
  - activity
  - settings

## Securite

Ce scope est un filtre UX pour le superadmin. La RLS reste la source de verite.
Un non-superadmin ne gagne aucun droit : s'il force un metadata `active_establishment_id`,
la RLS continue d'ignorer ce claim.

## Validation

- `npm run typecheck` OK.
- `npm run build` OK.

## Test attendu

1. Se connecter superadmin.
2. Creer au moins un etablissement.
3. Selectionner cet etablissement dans le switcher.
4. Cliquer `Ouvrir`.
5. Verifier que la sidebar affiche les pages etablissement.
6. Verifier que `/dashboard`, `/students`, `/companies`, `/classes` affichent les donnees de ce tenant uniquement.
7. Revenir sur `Vue globale` pour sortir de l'incarnation.
