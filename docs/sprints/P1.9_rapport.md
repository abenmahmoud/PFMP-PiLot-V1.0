- creation et lecture de periodes PFMP par classe ;

- vue professeur des placements visibles ;

## Migration

Les tables `pfmp_periods` et `placements` existaient deja dans `0001_init.sql`. J'ai donc fait une migration additive :

- extension `period_status` : `draft`, `published`, `cancelled` ;
- extension `stage_status` : `draft`, `confirmed`, `cancelled` ;
- `pfmp_periods.class_id`, `type`, `notes`, `archived_at` ;
- `placements.notes`, `archived_at` ;
- index utiles pour periodes, placements, archivage.

Decision : ne pas detruire/remplacer les enums historiques (`no_stage`, `found`, `pending_convention`, etc.) car ils sont encore utilises par les anciennes pages eleves/dashboard.

## Server functions

### `pfmpPeriods.functions.ts`

- `listPfmpPeriodsForEstablishment`
- `createPfmpPeriod`
- `updatePfmpPeriod`
- `archivePfmpPeriod`
- `restorePfmpPeriod`
- `publishPfmpPeriod`

Securite :
- `safeHandlerCall` partout ;
- `auth.getUser(accessToken)` via `getCallerProfile` ;
- service-role uniquement cote serveur ;
- isolation tenant via `resolveEstablishmentId` / `assertSameTenant` ;
- audit logs sur mutations.

### `placements.functions.ts`

- `listPlacementsForEstablishment`
- `listPlacementsForPeriod`
- `listPlacementsForStudent`
- `listPlacementsForReferent`
- `createPlacement`
- `updatePlacement`
- `updatePlacementStatus`
- `archivePlacement`
- `restorePlacement`

Permissions :
- admin/DDFPT/superadmin : CRUD tenant ;
- principal : creation/modification pour ses classes, pas archivage ;
- referent : lecture seule sur ses placements.

Note schema : `placements.referent_id` reference historiquement `teachers.id`. Le code conserve ce modele et synchronise `students.referent_id` avec `teachers.profile_id` quand disponible.

### `aiMatchmaking.functions.ts`

`suggestCompaniesForStudent` score les entreprises actives du tenant :

- compatibilite formation : 40 pts ;
- famille professionnelle : 20 pts ;
- entreprise pas encore faite par l'eleve : 15 pts ;
- fiabilite : 15 pts ;
- partenaire fort : 10 pts ;
- entreprise `to_avoid` : penalite -50 et exclue du top 3 si possible.

Aucun appel externe OpenAI/Claude : scoring deterministe, explicable, RGPD-friendly.

## UI livree

### Admin

- `/admin/pfmp-periods`
  - liste periodes ;
  - bouton nouvelle periode ;
  - calendrier synthetique ;
  - lien vers fiche periode.

- `/admin/pfmp-periods/$id`
  - metriques periode ;
  - liste placements ;
  - bouton `Affecter eleve` ;

- `/admin/placements`
  - liste globale ;
  - filtres statut/periode/classe ;
  - creation placement ;
  - changement rapide de statut.

- `/admin/placements/$id`
  - fiche placement ;
  - timeline ;
  - changement statut ;
  - entreprise/tuteur/referent.

- `/prof/placements/$id`
  - fiche lecture seule.

  - labels/tone ajoutes pour `draft`, `confirmed`, `cancelled`.
