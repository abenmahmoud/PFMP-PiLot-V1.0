## Dette demo restante

L'interface affiche encore :

- `Lycee Professionnel Jean Moulin`;
- des stats de demo ;
- des professeurs/entreprises/alertes de demo ;
- la zone sidebar `DEMO : CHANGER DE ROLE`.

Diagnostic : ce n'est plus un fallback Auth cause par RLS. Le profil reel est
charge, mais les pages metier utilisent encore les imports audites en P0.1
depuis `src/data/demo.ts`.

Action suivante recommandee :

1. P0.3 : dashboard Supabase reel, avec empty state prod si aucune donnee.
2. Auth strict prod : interdire le role switcher demo et le fallback demo hors
   `VITE_DEMO_MODE=true`.
3. P0.4+ : migration progressive des pages eleves, entreprises, visites,
   documents vers Supabase.
