# P0.5 — Plan Codex

## Objectif

Brancher le module entreprises en lecture Supabase, sans casser le mode demo.

## Strategie

- Suivre le pattern P0.4 : un service `src/services/companies.ts`, puis une route qui choisit `CompaniesDemo` ou `CompaniesSupabase` selon `isDemoMode()`.
- Garder la RLS naturelle : aucune requete ne force `establishment_id` cote client.
- Eviter le N+1 sur la liste : charger les entreprises, puis les tuteurs en une requete batchee avec `company_id in (...)`.
- Pour `buildCompanyIntelligence`, ne pas creer de vue SQL en P0.5. L'agregation JS suffit pour la lecture basique. Une vue/RPC pourra venir plus tard si le volume augmente.
- Creer `/companies/$id` pour la fiche entreprise : infos generales, tuteurs, historique placements.

## Fichiers

- `src/services/companies.ts`
- `src/routes/companies.tsx`
- `src/routes/companies.$id.tsx`
- `docs/sprints/P0.5_rapport.md`

## Validation prevue

- `npm run build`
- `npm run typecheck`
- Verification manuelle du mode vide Supabase : empty state propre.
- Verification manuelle du mode demo : liste entreprises toujours utilisable.
