# P0.6 — Plan Codex — My students Supabase

## Objectif

Brancher `/my-students` sur Supabase pour qu'un professeur referent ou principal voie une liste propre de ses eleves, sans revenir aux donnees demo en production.

## Strategie

- Garder le mode demo intact avec `isDemoMode()`.
- En mode Supabase, utiliser `useAuth()` au lieu de `useCurrentUser()`.
- Creer `src/services/myStudents.ts`.
- Ne pas filtrer par `establishment_id` cote client : la RLS reste la barriere de securite.
- Pour un referent : trouver son `teachers.id` via `profile_id`, puis filtrer `teacher_assignments.teacher_id`.
- Pour un principal : trouver son `teachers.id`, puis ses classes `classes.principal_id`.
- Charger les donnees associees en lots : students, classes, placements, companies, tutors, periods, alerts.
- Garder les boutons mobile-first : itineraire, appel, fiche, visite.

## Hors scope

- Creation reelle de visite.
- Validation de compte rendu.
- Tests RLS avec creation de comptes temporaires, a reprendre quand le MCP Supabase est stable.

## Validation

- `npm run typecheck`
- `npm run build`
- Preview Vercel sur `/my-students`.
