# P1.5 - Plan Codex - Portail eleve sans email

Date : 12 mai 2026
Branche : `feature/p1-5-student-portal-no-email`

## Objectif

Livrer un portail public `/eleve` permettant a un eleve de s'identifier avec un code PFMP/QR sans compte Supabase, puis d'acceder a un dashboard lecture seule `/eleve/dashboard`.

## Plan d'execution

1. Server functions
   - Creer `src/server/studentPortal.functions.ts`.
   - Copier la logique `normalizeCode()` et `hashAccessCode()` depuis P1.4 pour eviter le couplage.
   - Ajouter `validateStudentCode()` avec service-role, validation statut, update `last_used_at`, audit log `student_portal.code_validated`, retour sanitise.
   - Ajouter `getStudentPublicDashboard()` avec verification code actif, lecture student/class/placement/company/tutor, retour sanitise.

2. Route publique `/eleve`
   - Remplacer le stub P1.4.
   - Gerer route parent avec `useMatchRoute` + `<Outlet />` pour laisser `/eleve/dashboard` se rendre.
   - Formulaire public sans `AppLayout`, prefill `?code=`, auto-submit, messages d'erreur FR.
   - Sauvegarder une session legere dans `sessionStorage`.

3. Route publique `/eleve/dashboard`
   - Creer `src/routes/eleve.dashboard.tsx`.
   - Lire `sessionStorage`, redirect `/eleve` si absent.
   - Charger le dashboard public via server function.
   - Afficher stage, entreprise, tuteur, infos eleve en lecture seule et gerer null partout.

4. Documentation et validation
   - Ecrire `docs/sprints/P1.5_rapport.md`.
   - Lancer `npm run typecheck`, `npm run build`, `git diff --check`.

## Contraintes suivies

- Zero migration SQL.
- Zero `auth.signIn()` pour l'eleve.
- Zero `AppLayout` admin sur `/eleve/*`.
- `sessionStorage` uniquement.
- Tous les retours server functions passent par `sanitizeForSeroval()`.
- Dashboard eleve lecture seule.
