# P2.5 - Commercialisation durable + onboarding multi-etablissement

Date : 15 mai 2026  
Branche : `feature/p2-5-commercial-onboarding`  
Objectif : transformer le MVP technique PFMP Pilot AI en produit presentable et vendable a plusieurs etablissements, sans fragiliser les portails metier.

## 1. Positionnement

P2.5 n'est pas une simple landing page. Le sprint pose une couche go-to-market durable :

- page publique commerciale sur `/` ;
- page d'acces aux 3 portails sur `/portails` ;
- tunnel `Demander une demo / devis` persistant cote DB ;
- page securite/RGPD publique ;
- runbook d'onboarding multi-etablissement ;
- conservation de tous les chemins metier existants (`/admin/*`, `/prof/*`, `/eleve`).

## 3. Architecture cible

### Routes publiques

- `/` : landing commerciale publique.
- `/portails` : ancienne page de choix des espaces.
- `/devis` : formulaire public demande demo/devis.
- `/securite-rgpd` : page publique de confiance/securite/RGPD.

### Server / DB

- Migration `sales_leads` pour stocker les demandes commerciales.
- Server function `submitSalesLead` avec validation stricte et `safeHandlerCall`.
- Aucun stockage de donnees sensibles inutiles.
- Audit log optionnel si rattachement tenant absent : la table `sales_leads` fait foi.

### Onboarding

- Documentation operationnelle `docs/ops/PILOT_ONBOARDING_RUNBOOK.md`.
- Checklist durable pour creer un etablissement pilote :
  1. creer tenant ;
  2. inviter admin/DDFPT ;
  3. importer classes/eleves ;
  4. creer profs ;
  5. creer entreprises/tuteurs ;
  6. creer periode PFMP ;
  7. creer placement ;
  8. generer codes eleves ;
  9. tester portail eleve ;
  10. tester signature.

## 5. Decisions techniques

- Garder `/admin/login`, `/prof/login`, `/eleve` inchanges pour ne pas casser les usages.
- Deplacer l'ancien choix des portails vers `/portails`, pour permettre a `/` de devenir commercial.
- Ne pas ajouter de dependance UI.
- Ne pas brancher d'envoi email obligatoire dans P2.5 : le lead DB suffit pour ne pas bloquer le flux commercial.
- Ne pas faire de paiement/pricing complexe : le devis reste humain, adapte aux lycees/groupes/rectorats.
- Garder un design premium mais sobre, compatible avec l'identite existante et sans hero marketing creux.
