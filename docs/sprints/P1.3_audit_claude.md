## 2. Parseur src/services/imports/siecle.ts

### Architecture
Parseur pur client-side, zero side-effect, zero Supabase. Correct.

### Points forts
- Import dynamique xlsx (await import('xlsx')) -- chunk Excel (~429 KB gzip 143 KB) charge uniquement pour XLS/XLSX, pas pour CSV.
- Detection encodage UTF-8 / latin1 avec TextDecoder fatal mode + fallback.
- Detection separateur ; / , sur la premiere ligne, hors guillemets.
- Parser CSV maison avec gestion des guillemets et CRLF. Robuste.
- Normalisation headers : stripAccents().toUpperCase().replace(/[^A-Z0-9]/g, '') -> matching NOM_ELEVE = NOMELEVE = NOM ELEVE.
- Dedoublonnage par cle lastName|firstName|divisionName normalisee.
- Erreurs non bloquantes dans errors[], import continue.
- extractLevel() gere CAP1, CAP2, 1, 2, 3 -> N/C si non detecte.
- titleCase() avec regex Unicode \p{L} pour les prenoms composes et accents.
- Preview = 10 premieres lignes brutes.

### Validation sur seed fictif (resultat rapport Codex)
{ "classes": 3, "students": 15, "preview": 10, "errors": 0, "classNames": ["1MSPC", "2MVA", "CAP1EPC"] }
Conforme au brief : 3 classes, 15 eleves.

### Risques NEGLIGEABLES
- Parser CSV maison : cas edge (cellules vides en fin de ligne, guillemets imbriques) geres. Acceptable pour SIECLE qui produit des CSV propres.
- inferHeaders() utilise pour XLSX : reconstruit les headers depuis les cles du premier objet. Correct.
