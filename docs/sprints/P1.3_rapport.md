Date : 12 mai 2026

## Objectif livre

Un admin, DDFPT ou superadmin peut ouvrir `/imports/siecle`, charger un export SIECLE CSV/XLS/XLSX, verifier les donnees detectees, selectionner les divisions, lancer un dry-run obligatoire, puis confirmer l'import reel des classes et eleves du tenant actif.

Le sprint ne cree aucun compte eleve et n'envoie aucun email eleve.

## Format SIECLE supporte

Colonnes principales reconnues :

- `NOM_ELEVE`, `NOM`, `NOMFAMILLE` -> nom eleve
- `PRENOM_ELEVE`, `PRENOM` -> prenom eleve
- `DIVISION`, `CLASSE` -> classe/division
- `CODE_MEF`, `MEF` -> code formation
- `LIBELLE_MEF`, `LIBELLE`, `FORMATION` -> libelle formation prefere
- `EMAIL_ELEVE`, `EMAIL`, `MAIL` -> email de contact optionnel

Formats :

- CSV avec separateur auto-detecte `;` ou `,`
- XLS/XLSX via `xlsx`
- UTF-8 avec fallback `iso-8859-1`

Le chargement `xlsx` est dynamique : le chunk principal `/imports/siecle` reste leger pour le cas CSV, et le parseur Excel n'est charge que si l'utilisateur importe un fichier `.xls` ou `.xlsx`.

## Donnees volontairement ignorees

Conformement au brief, le sprint ignore et ne stocke pas :

- INE
- date de naissance
- sexe
- responsables legaux
- adresse
- regime / boursier
- tout code eleve ou QR code

Le fichier importe n'est jamais stocke en base ni dans Supabase Storage.

## Strategie doublons

Au parsing :

- dedoublonnage par `lastName + firstName + divisionName`
- doublons signales dans `errors[]` sans bloquer l'import

En base :

- une classe est reutilisee si une classe du meme tenant a deja le meme `name`
- un eleve est ignore si un eleve actif existe deja avec meme `first_name + last_name + class_id`
- les doublons dans le meme batch sont aussi ignores

## Securite et permissions

La server function `importSiecleData` :

- utilise une service-role key uniquement cote serveur
- verifie d'abord `auth.getUser(accessToken)`
- lit le profil appelant
- autorise uniquement `admin`, `ddfpt`, `superadmin`
- refuse un admin/DDFPT hors de son tenant
- injecte `establishment_id` cote serveur
- impose `dryRun: true` sans ecriture
- ecrit `audit_logs.action = "import.siecle.students"` uniquement lors de l'import reel
