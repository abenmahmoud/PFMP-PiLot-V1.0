# P1.7 — Architecture 3 portails séparés — Plan Codex

> Branche : `feature/p1-7-three-portals`  
> Objectif : séparer visuellement et fonctionnellement les entrées Administration, Professeur et Élève sans modifier la base.

## Stratégie

Migrer l'UX vers trois portails tout en gardant les routes historiques en redirection ou compatibilité. Le risque principal est TanStack Router : les routes plates avec préfixes peuvent devenir des layouts parents. Je vais donc créer des layouts `/admin` et `/prof` explicites avec `<Outlet />`, puis ajouter les nouvelles routes préfixées progressivement.

## Étapes

1. **Permissions et redirections**
   - Mettre à jour `getHomePathForRole`.
   - Ajouter `getLoginPathForRole`.
   - Ajouter des helpers de groupe : `isAdminPortalRole`, `isProfPortalRole`.

2. **Layouts et sidebars**
   - Créer `src/layouts/AppLayoutAdmin.tsx`.
   - Créer `src/layouts/AppLayoutProf.tsx`.
   - Créer `src/components/sidebars/SidebarAdmin.tsx`.
   - Créer `src/components/sidebars/SidebarProf.tsx`.
   - Garder `AppLayout` et `Sidebar` comme compatibilité si certaines routes anciennes les importent encore.

3. **Entrée publique**
   - Refactorer `/` en page d'orientation avec trois cartes : Administration, Professeur, Élève.
   - Créer `/admin/login` et `/prof/login` avec validation de rôle après login.
   - Garder `/login` comme redirection/compatibilité vers `/`.

4. **Routes parent**
   - Créer `src/routes/admin.tsx` comme layout parent `/admin` avec `RoleGuard`.
   - Créer `src/routes/prof.tsx` comme layout parent `/prof` avec `RoleGuard`.

5. **Routes admin préfixées**
   - Ajouter des wrappers `/admin/*` pour les routes métier existantes.
   - Approche privilégiée : exporter les composants page existants quand possible, ou créer des wrappers qui réutilisent la même logique.
   - Mettre à jour les liens internes critiques vers `/admin/*`.

6. **Routes prof**
   - Créer `/prof/dashboard`.
   - Créer `/prof/my-classes`.
   - Créer `/prof/my-students`.
   - Créer des stubs lisibles pour `/prof/visits`, `/prof/assignments`, `/prof/alerts`, `/prof/companies`, `/prof/pfmp-periods` si nécessaire.

7. **Backward compatibility**
   - Les anciennes URL principales (`/dashboard`, `/classes`, `/students`, etc.) redirigent vers `/admin/...`.
   - Les routes détail critiques (`/classes/$id`, `/students/$id`, `/teachers/$id`, etc.) redirigent vers leurs équivalents `/admin/...`.

8. **Validation**
   - `npm run typecheck`
   - `npm run build`
   - Rapport `docs/sprints/P1.7_rapport.md`

## Points d'attention

- Zéro migration SQL.
- Ne pas toucher aux server functions.
- Les routes `/eleve` et `/eleve/dashboard` restent fonctionnelles.
- Ne pas casser le mode démo : les routes admin/prof doivent rester navigables avec les profils démo.
- Les anciennes pages ne sont pas supprimées sans redirection.

## Livrable attendu

PR prête pour audit avec :
- portail `/` clair,
- login admin/prof séparé,
- layout admin/prof distinct,
- routes `/admin/*` et `/prof/*` opérationnelles,
- anciennes URL redirigées,
- typecheck/build verts.
