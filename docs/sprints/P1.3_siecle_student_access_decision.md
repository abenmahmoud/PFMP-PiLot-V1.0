# P1.3 - Decision produit : SIECLE d'abord, codes eleves par classe

## Decision

PFMP Pilot AI ne doit pas inviter les eleves par email pour le MVP terrain.

Les eleves de lycee professionnel et de CAP peuvent avoir des difficultes d'acces email, de creation de compte, de mot de passe oublie, ou d'usage regulier d'une boite mail scolaire. L'objectif produit est d'aller vite en etablissement sans transformer le DDFPT ou le professeur principal en support technique.

Donc :

- les adultes gardent des comptes nominatifs ;
- les eleves sont crees par import administratif ;
- les acces eleves sont generes par classe, par le professeur principal ou l'administration ;
- chaque eleve a son code personnel, jamais un code partage par toute la classe.

## Source officielle prioritaire

L'import eleves/classes doit cibler SIECLE en priorite.

Pronote peut rester une compatibilite future, mais la source administrative attendue pour creer proprement les classes, divisions et eleves est SIECLE.

## Flux cible

1. Admin/DDFPT importe un export SIECLE.
2. L'application detecte les divisions/classes.
3. L'application cree ou met a jour :
   - les classes ;
   - les eleves ;
   - le rattachement eleve -> classe.
4. Depuis une classe, le professeur principal ou l'admin genere les acces eleves.
5. L'application produit :
   - une liste imprimable des codes ;
   - un QR code par eleve ;
   - une option de reinitialisation/revocation.
6. L'eleve entre dans l'application via "Eleve" puis son code.

## Regle de securite

Un code classe partage est interdit.

Chaque eleve doit avoir un code unique rattache a son `student_id`, idealement stocke hashé cote base. L'interface peut generer/imprimer les codes en clair une seule fois, puis ne plus les afficher completement.

## Consequence roadmap

Avant les entreprises, periodes PFMP et documents, il faut stabiliser :

1. P1.3 - Import SIECLE classes + eleves.
2. P1.4 - Generation des codes eleves par classe + QR/PDF imprimable.
3. P1.5 - Portail d'entree : Administration / Enseignant / Eleve.
4. P1.6 - Espace eleve par code, sans email.

Les imports Pronote/STSWeb deviennent secondaires et seront traites comme formats compatibles, pas comme source produit principale.
