# P2.8 — Convention PDF + signatures multi-signataires

## Résultat
Le workflow convention est maintenant raccordé de bout en bout :

- Une convention `draft` peut générer un PDF réel depuis le modèle de classe.
- Le PDF est stocké dans `generated-pdfs` et relié à `documents.generated_document_id`.
- Le bouton `Envoyer pour signatures` crée les signatures tuteur, parent si mineur, et DDFPT.
- Les liens magiques P2.4 sont réutilisés.
- Quand toutes les signatures sont collectées, le PDF final signé est généré et la convention passe `signed`.
- Le dossier PFMP passe `signed_convention`.

## Décisions
- Les PDF restent sobres et natifs `pdf-lib`, avec markdown simple et placeholders de signature.
- Le moteur P2.4 reste la source unique pour tokens, emails, hash, preuve et finalisation.
- Les statuts `generated`, `pending_signatures`, `signed` ont été ajoutés à `document_status` pour éviter un workflow parallèle.
- Le DDFPT est signataire obligatoire; si l'élève est mineur, l'email parent devient obligatoire avant envoi.
