# P1.2 — Invitations admin/DDFPT et cockpit utilisateurs tenant — Rapport Codex

## Objectif

Transformer la création d'établissement en vrai flux SaaS : le superadmin crée un tenant, invite son premier admin/DDFPT, puis l'admin/DDFPT peut inviter les utilisateurs de son établissement.

## Fichiers créés

- `src/server/invitations.functions.ts`
  - Fonction serveur TanStack Start `inviteUserToEstablishment`.
  - Utilise `SUPABASE_SERVICE_ROLE_KEY` côté serveur uniquement.
  - Revalide le JWT appelant via Supabase Auth.
  - Vérifie les droits :
    - superadmin : peut inviter `admin`, `ddfpt`, `principal`, `referent`, `eleve` ;
    - admin/DDFPT : peut inviter uniquement `principal`, `referent`, `eleve` dans son tenant.
  - Appelle `supabase.auth.admin.inviteUserByEmail`.
  - Upsert `profiles`.
  - Crée/met à jour une ligne `teachers` pour les rôles pédagogiques `principal` et `referent`.
  - Écrit `audit_logs` avec `user.invited`.

- `src/components/InviteUserForm.tsx`
  - Formulaire réutilisable d'invitation.
  - Utilisé par le superadmin et par le cockpit admin tenant.

- `src/routes/onboarding.tsx`
  - Page post-magic-link.
  - Affiche le compte rattaché.
  - Permet de définir le mot de passe.
  - Redirige ensuite vers `/dashboard`.

- `src/routes/admin.users.tsx`
  - Cockpit utilisateurs tenant.
  - Liste les profils du tenant.
  - Permet à l'admin/DDFPT d'inviter les rôles autorisés.
  - Superadmin peut l'utiliser avec un tenant actif.

- `src/services/tenantUsers.ts`
  - Lecture des profils du tenant courant.

## Fichiers modifiés

## Variables à configurer dans Vercel

Obligatoires pour que l'invitation fonctionne en production :

```env
SUPABASE_SERVICE_ROLE_KEY=...
PFMP_APP_URL=https://www.pfmp-pilot.fr
```

Ces variables sont serveur uniquement. Ne jamais les préfixer par `VITE_`.

## Ce qui reste hors scope

- Statut détaillé des invitations (`pending`, `accepted`, `expired`) via table dédiée.
- Renvoyer une invitation expirée.
- Désactiver/archiver un utilisateur.
- Templates email Supabase personnalisés.
- Magic-links tuteurs entreprise.

## Tests attendus en prod

1. Superadmin crée un établissement avec email DDFPT renseigné.
2. Le DDFPT reçoit l'invitation Supabase.
3. Le DDFPT ouvre `/onboarding`, définit son mot de passe.
4. Le DDFPT arrive sur `/dashboard`.
5. Le DDFPT ouvre `/admin/users`.
6. Il invite un `referent`.
7. Le superadmin voit le compte dans `/superadmin/establishments/$id`.
