# P1.2.2 - Plan d'attaque

## Objectif

Ajouter une brique SaaS visible et exploitable : chaque etablissement dispose de son lien officiel et de son QR code pour acceder a son espace tenant.

Le superadmin pourra remettre ce lien au lycee. L'admin/DDFPT pourra le retrouver depuis `/settings`.

## Lecture du brief

Brief lu : `docs/briefs/P1.2.2_tenant_access_link_qr.md`.

Contraintes retenues :

- aucune migration SQL ;
- aucune policy RLS ;
- pas de configuration DNS automatisee dans ce sprint ;
- mode demo intact ;
- logique URL centralisee ;
- composant reutilisable superadmin + tenant settings ;
- custom domain verifie prioritaire si present.

## Branche

Branche de travail : `codex/p1-2-2-tenant-access-link-qr`.

Cette branche est empilee sur P1.2.1 parce que PR #25 n'est pas encore mergee. Apres merge de #25, elle pourra etre rebasee sur `main` si necessaire.

## Surfaces concernees

### 1. `src/lib/tenantAccess.ts`

Creer un helper dedie pour eviter de disperser la logique des URLs tenant.

Fonctions prevues :

- `getTenantAccessSlug(establishment)` : retourne `subdomain || slug`.
- `buildTenantPlatformUrl(establishment)` : construit `https://<subdomain-ou-slug>.<VITE_PLATFORM_DOMAIN>`.
- `buildTenantAccessUrl(establishment)` : retourne le custom domain verifie si disponible, sinon l'URL plateforme.
- `getTenantAccessStatus(establishment)` : donne le libelle/badge a afficher.

### 2. `src/components/TenantAccessCard.tsx`

Creer un composant pur et reutilisable.

Contenu prevu :

- URL principale lisible et selectionnable ;
- badge "Sous-domaine plateforme" ou "Domaine personnalise verifie" ;
- QR code contenant l'URL principale ;
- bouton copier ;
- bouton ouvrir ;
- bouton telecharger QR si la librairie choisie expose un canvas proprement ;
- fallback propre si le navigateur refuse le clipboard.

Decision librairie :

- Installer `qrcode.react` si disponible, pour eviter une implementation QR maison.
- Utiliser `QRCodeCanvas` afin de permettre le telechargement PNG.
- Si l'installation echoue, revenir a un QR SVG sans download dans ce sprint plutot que d'ajouter un service tiers externe.

### 3. `src/routes/superadmin.establishments.$id.tsx`

Ajouter `TenantAccessCard` dans la fiche detail superadmin, en haut de page apres les badges.

Objectif UX : BraveHeart voit immediatement le lien a transmettre au lycee.

La carte "Domaines" existante reste en place pour les champs techniques.

### 4. `src/routes/settings.tsx`

Ajouter `TenantAccessCard` dans les settings Supabase, probablement sous la carte "Identite de l'etablissement".

Objectif UX : le DDFPT/admin peut copier le lien de son espace sans repasser par le superadmin.

En mode demo, afficher la carte uniquement si elle n'alourdit pas l'UX ; sinon rester invisible/demo-safe.

### 5. `docs/sprints/P1.2.2_rapport.md`

Documenter :

- fichiers modifies ;
- choix QR ;
- fonctionnement URL ;
- validations ;
- reste a verifier cote DNS wildcard Vercel/OVH.

## Risques identifies

- Le wildcard `*.pfmp-pilot.fr` doit etre correctement configure cote Vercel/DNS pour que tous les liens soient vraiment accessibles en prod.
- Le QR code peut necessiter une dependance. Je privilegie une librairie locale plutot qu'un service externe pour eviter de fuiter les URLs a un tiers.
- Les routes superadmin detail et settings sont deja modifiees par P1.2.1 ; la PR P1.2.2 restera empilee jusqu'au merge de #25.
