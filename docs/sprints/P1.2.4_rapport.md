## 3. Migration SQL

Fichier cree :

- `supabase/migrations/20260512153000_add_archived_at_to_profiles.sql`

Contenu :

- `ALTER TABLE public.profiles ADD COLUMN IF NOT EXISTS archived_at TIMESTAMPTZ NULL`
- index partiel `idx_profiles_archived_at`
- commentaire colonne

Decision : je n'ai pas ajoute de filtre RLS global sur `profiles.archived_at`.

Raison : `fetchProfile()` doit continuer a lire le profil courant. Un filtre global RLS sur les profiles archives pourrait casser l'auth et rendre le diagnostic plus difficile. Les listes et server functions filtrent explicitement les archives.

`supabase migration list` : non execute depuis cette session Codex. La migration est versionnee dans le repo et doit etre appliquee/controlee par Claude ou via le process Supabase habituel avant test prod complet.

## 7. Decisions techniques

### Soft-delete vs hard-delete

Soft-delete (`profiles.archived_at`) est le mode normal : reversible, auditable, operationnellement sur. Hard-delete est conserve en danger zone superadmin pour corriger les comptes de test ou les invitations accidentelles definitivement.

### Email pending only

`updateUserEmail` refuse les users confirmes. Changer l'email d'un compte confirme cote admin serait une prise de controle potentielle. Pour un compte confirme, le reset password reste disponible.

### Audit log avant cascade delete

Le hard-delete ecrit `user.hard_deleted` avant suppression, avec snapshot dans `metadata`. Ensuite il supprime les logs lies a l'utilisateur cible, les roles, le profile, puis `auth.users`.

### Resend invitation

Implementation :

1. tentative `auth.admin.inviteUserByEmail(email, { redirectTo })` ;
2. fallback `auth.resend({ type: 'signup', email })` si Supabase refuse le premier appel.

Point a auditer en preview : selon le comportement exact Supabase Auth pour un user deja invite, le fallback peut etre celui qui envoie effectivement l'email.

### RLS archived_at

Pas de changement RLS global pour eviter un effet de bord auth. Filtrage explicite dans les server functions/listes.
