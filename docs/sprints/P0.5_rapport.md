## Decisions techniques

- Aggregation JS pour P0.5 au lieu d'une vue SQL/RPC. C'est suffisant pour cette phase lecture et evite une migration prematuree.
- Liste sans N+1 : les tuteurs sont charges en batch pour les entreprises visibles.
- Detail entreprise : placements + students + periodes + tuteurs charges par lots.
- Mode demo conserve : `VITE_DEMO_MODE=true` continue d'utiliser `src/data/demo.ts`.
- En mode Supabase, les boutons de creation restent desactives : mutations prevues plus tard.
