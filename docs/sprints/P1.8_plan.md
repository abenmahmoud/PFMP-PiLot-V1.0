# P1.8 — Référentiel entreprises + tuteurs

Date : 13 mai 2026  
Branche : `feature/p1-8-companies-tutors`

## Objectif

Rendre le référentiel entreprises réellement exploitable par un admin/DDFPT avant les affectations PFMP : création, édition, archivage/restauration, tuteurs associés, import CSV/XLSX, et consultation lecture seule côté prof.

## Audit initial

Tables déjà disponibles :

- `companies` : `id`, `establishment_id`, `name`, `address`, `city`, `zip_code`, `phone`, `email`, `website`, `siret`, `siren`, `sector`, `professional_family`, `compatible_formations`, `students_hosted`, `last_hosted_at`, `reliability`, `status`, `internal_notes`, `history`, `archived_at`, `created_at`, `updated_at`.
- `tutors` : `id`, `establishment_id`, `company_id`, `first_name`, `last_name`, `function`, `email`, `phone`, `responsiveness`, `internal_notes`, `archived_at`, `created_at`, `updated_at`.
- Index unique déjà présent : `(establishment_id, siret) WHERE siret IS NOT NULL`.

Code existant :

- `src/services/companies.ts` lit déjà les entreprises/tuteurs via RLS.
- `src/routes/companies.tsx` affiche déjà la liste en lecture, mais le bouton Ajouter est désactivé.
- `src/routes/companies.$id.tsx` affiche déjà la fiche entreprise et ses tuteurs, sans actions de mutation.
- P1.7 expose déjà `/admin/companies`, `/admin/companies/$id`, `/prof/companies`, `/prof/companies/$id`.
- `src/server/_lib.ts` fournit `createAdminClient`, `safeHandlerCall`, `getCallerProfile`, `insertAuditLog`, `validateUuid`.

## Plan d'implémentation

### 1. Server functions

Créer `src/server/companies.functions.ts` avec :

- `listCompaniesForEstablishment({ accessToken, includeArchived? })`
- `createCompany({ accessToken, data })`
- `updateCompany({ accessToken, companyId, data })`
- `archiveCompany({ accessToken, companyId })`
- `restoreCompany({ accessToken, companyId })`
- `createTutor({ accessToken, companyId, data })`
- `updateTutor({ accessToken, tutorId, data })`
- `archiveTutor({ accessToken, tutorId })`
- `restoreTutor({ accessToken, tutorId })`
- `importCompanies({ accessToken, rows })`

Règles :

- mutations réservées à `admin`, `ddfpt`, `superadmin`.
- `principal`/`referent` : lecture seule via services existants.
- superadmin doit fournir/obtenir un tenant actif ; sinon erreur explicite.
- `establishment_id` toujours injecté serveur.
- audit log sur chaque mutation.
- `safeHandlerCall` sur tous les handlers.
- validation SIRET simple : vide ou 14 chiffres.
- import : max 200 lignes, déduplication par SIRET si présent, sinon `name + city`.

### 2. Types/services client

Étendre `src/services/companies.ts` :

- types `CompanyCreateInput`, `CompanyUpdateInput`, `TutorCreateInput`, `CompanyImportRow`.
- helper de parse CSV/XLSX côté client pour l'import entreprises, en mémoire uniquement.
- éventuellement option `includeArchived` dans `fetchCompanies`.

### 3. Composants UI

Créer `src/components/companies/` :

- `CompanyFormModal.tsx`
- `TutorFormModal.tsx`
- `CompanyImportModal.tsx`

Design sobre, dense, SaaS opérationnel. Pas de landing page, pas de décor inutile.

### 4. Route `/admin/companies`

Modifier `src/routes/companies.tsx` :

- bouton Ajouter actif uniquement admin/DDFPT/superadmin.
- bouton Import CSV/XLSX actif uniquement admin/DDFPT/superadmin.
- filtres existants conservés.
- affichage des entreprises archivées via toggle ou filtre simple.
- mutation successful -> reload liste.
- `/prof/companies` reste lecture seule : pas de bouton mutation.

### 5. Route `/admin/companies/$id`

Modifier `src/routes/companies.$id.tsx` :

- actions Modifier / Archiver / Restaurer entreprise côté admin.
- ajout/modification/archivage tuteur côté admin.
- côté prof : aucune action mutation, consultation seulement.

### 6. Rapport

Créer `docs/sprints/P1.8_rapport.md` avec :

- fichiers modifiés/créés ;
- sécurité/permissions ;
- stratégie import/dédoublonnage ;
- tests `typecheck`, `build`, smoke local ;
- limites restantes pour P1.9 placements.
