# P2.8 — Convention PDF + signatures multi-signataires

## Objectif
Brancher le workflow convention bout-en-bout : convention logique `documents`, PDF `generated_documents`, signatures P2.4, puis archivage du PDF final signé.

## Plan d'attaque
1. Ajouter la migration additive pour les champs parent/naissance, le lien `documents.generated_document_id` et les nouveaux statuts de convention.
2. Créer `src/lib/pdfConvention.ts` pour générer un PDF A4 depuis le modèle markdown affecté à la classe.
3. Créer `src/server/conventions.functions.ts` avec génération PDF, envoi signatures, liste validation et recompute de statut.
4. Réutiliser le moteur P2.4 pour les signatures, puis marquer automatiquement la convention et le placement comme signés quand toutes les signatures sont collectées.
5. Étendre la fiche document avec un workflow lisible : compléter, générer PDF, envoyer signatures, suivre, télécharger/vérifier.
6. Étendre la liste documents avec un filtre `A valider` et supprimer les boutons morts.
7. Valider par `npm run typecheck` et `npm run build`.

## Garde-fous
- Migration strictement additive.
- Aucun doublon du moteur signatures.
- `safeHandlerCall` sur toutes les nouvelles server functions.
- Isolation tenant systématique via le profil appelant.
- DDFPT/admin/superadmin seulement pour les actions de génération et signature.
