# P1.0 — Correctif route creation etablissement

## Symptome

Sur `/superadmin/establishments/new`, l'application affichait encore la liste des etablissements au lieu du formulaire de creation.

## Cause racine

TanStack Router considere `superadmin.establishments.tsx` comme route parent de `superadmin.establishments.new.tsx`.
La route parent rendait toujours son contenu de liste et ne rendait jamais l'`Outlet`, donc la route enfant etait bien matchee mais jamais affichee.

## Correction

- `src/routes/superadmin.establishments.tsx` importe maintenant `Outlet` et `useRouterState`.
- La liste reste affichee uniquement sur le chemin exact `/superadmin/establishments`.
- Les chemins enfants, dont `/superadmin/establishments/new`, rendent l'`Outlet`.

## Validation

- `npm run typecheck` OK.
- `npm run build` OK.

## Suite

Tester en production apres merge :

1. Ouvrir `/superadmin/establishments`.
2. Cliquer sur `Nouvel etablissement`.
3. Verifier que `/superadmin/establishments/new` affiche le formulaire.
4. Creer un etablissement de test.
