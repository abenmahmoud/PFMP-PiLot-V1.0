## Decision QR code

J'ai ajoute `qrcode.react` pour generer le QR code localement dans l'application.

Raison : eviter un service externe de generation QR qui recevrait les URLs des lycees. Le QR contient uniquement le lien public du tenant.

Le composant utilise `QRCodeCanvas`, ce qui permet de telecharger le QR en PNG via le canvas.

## Comportement URL

La logique est centralisee dans `src/lib/tenantAccess.ts` :

- si `custom_domain` existe et `domain_verified=true`, le domaine personnalise devient le lien principal ;
- sinon, le lien principal est `https://<subdomain-ou-slug>.<VITE_PLATFORM_DOMAIN>` ;
- le sous-domaine utilise `subdomain || slug`.

## UX livree

- Superadmin : carte visible en haut de `/superadmin/establishments/$id`.
- Admin/DDFPT : carte visible dans `/settings`.
- Actions disponibles :
  - copier le lien ;
  - ouvrir l'espace ;
  - telecharger le QR code ;
  - selectionner manuellement l'URL si le clipboard est bloque.
