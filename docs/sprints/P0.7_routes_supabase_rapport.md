## Decisions techniques

Chaque route suit le pattern deja valide sur les sprints precedents :

- `isDemoMode()` garde le rendu historique base sur `@/data/demo`.
- Le mode Supabase passe par `useAuth()`.
- Les lectures sont isolees dans `src/services/*`.
- Les empty states sont explicites pour un tenant vide.
- Les actions non branchees en mutation (`Ajouter`, `Nouvelle classe`, `Televerser`, `Export ZIP`) sont desactivees en mode Supabase.

Les services ne recoivent pas `establishment_id` depuis le client. La RLS Supabase reste la source de verite pour l'isolation multi-tenant.
