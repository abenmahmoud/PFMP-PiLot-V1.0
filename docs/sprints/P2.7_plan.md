# P2.7 — Document Intelligence PFMP

## Objectif

Transformer les conventions et livrets fournis par chaque lycee en modeles numeriques exploitables par classe, avec import DOCX/PDF, analyse IA assistee, validation humaine et preparation du workflow signature.

## Contexte

P2.6 ajoute deja le studio conventions par classe :

- une bibliotheque de `document_templates` ;
- une affectation classe -> modele ;
- une creation automatique des conventions eleves lors de la preparation d'une periode PFMP.

Le manque actuel est le document source : un lycee peut fournir un `.docx`, un PDF remplissable, un PDF plat ou un scan papier. Il faut donc conserver le fichier source, detecter ses champs, expliquer ce qui a ete reconnu, puis laisser le superadmin/DDFPT valider avant affectation.

## Contraintes

- Migration additive uniquement.
- Aucun bouton mort en production.
- L'IA propose, l'humain valide.
- Le service role reste uniquement cote server functions.
- Tenant isolation stricte.
- Pas de dependance lourde : DOCX/PDF complets et OCR avance seront branches plus tard ; ce sprint pose le modele durable et l'analyse IA/heuristique.
- `npm run typecheck` et `npm run build` doivent rester verts.

## Livrables

1. Schema `document_template_fields` pour stocker les champs detectes et leur mapping.
2. Extension `document_templates` pour source DOCX/PDF/scan, chemin Storage, statut d'analyse, nombre de champs et revue humaine.
3. Server fn `analyzeConventionTemplateSource` :
   - recoit metadonnees + contenu base64 optionnel + texte extrait/copie ;
   - upload best-effort du fichier source dans `documents-private` ;
   - cree un template brouillon ;
   - genere un mapping IA si `ANTHROPIC_API_KEY` existe, sinon heuristique ;
   - cree les champs detectes.
4. Server fn `listDocumentTemplateFields`.
5. UI `/admin/documents` :
   - panneau "Import intelligent DOCX/PDF" ;
   - etapes source -> analyse -> revue -> affectation classe ;
   - visualisation des champs detectes.
6. Rapport `docs/sprints/P2.7_rapport.md`.

## Criteres d'acceptation

- [ ] Un DDFPT/superadmin peut enregistrer une source DOCX/PDF comme modele de convention.
- [ ] Le modele garde son nom de fichier, type source, statut d'analyse et champs detectes.
- [ ] Les champs essentiels eleve/periode/entreprise/tuteur/signatures apparaissent en revue.
- [ ] Un modele importe peut etre affecte a une classe.
- [ ] La preparation des conventions continue d'utiliser le modele affecte a la classe.
- [ ] Build et typecheck OK.
