# P1.4 - Plan Codex - Codes eleves par classe

Date : 12 mai 2026  
Branche : `feature/p1-4-student-access-codes`  
Objectif : permettre a un admin, DDFPT, professeur principal ou superadmin de generer des codes personnels eleves par classe, sans email eleve ni compte `auth.users`.

## 1. Lecture produit

Decision source : `docs/sprints/P1.3_siecle_student_access_decision.md`.

Le MVP terrain doit eviter l'invitation email pour les eleves. Les adultes gardent des comptes nominatifs, mais les eleves importes depuis SIECLE recoivent un code personnel remis par l'etablissement. Un code partage par classe est interdit.

P1.4 couvre uniquement :

- generation / regeneration de codes eleves par classe ;
- stockage hash cote base ;
- affichage du statut des codes ;
- fiche classe imprimable avec code et QR au moment de generation ;
- revocation d'un code.

P1.4 ne couvre pas encore :

- connexion eleve effective ;
- espace eleve complet ;
- signature eleve ;
- stockage de code en clair.

## 2. Schema SQL prevu

Migration : `supabase/migrations/20260512190000_student_access_codes.sql`

Table : `student_access_codes`

- `id uuid primary key default gen_random_uuid()`
- `establishment_id uuid not null references establishments(id) on delete cascade`
- `student_id uuid not null references students(id) on delete cascade`
- `code_hash text not null`
- `code_hint text not null`
- `status text not null check in ('active','revoked','expired')`
- `expires_at timestamptz null`
- `last_used_at timestamptz null`
- `created_by uuid null references profiles(id) on delete set null`
- `revoked_at timestamptz null`
- timestamps

Contraintes :

- un seul code actif par eleve : index unique partiel `(student_id) where status = 'active'`;
- index tenant/classe indirect via `student_id`;
- RLS activee ;
- SELECT tenant-scoped pour superadmin/admin/ddfpt/principal/referent selon policies existantes ;
- INSERT/UPDATE reserves superadmin/admin/ddfpt/principal, avec tenant match.

## 3. Types et services

`src/lib/database.types.ts`

- ajouter `StudentAccessCodeStatus`
- ajouter `StudentAccessCodeRow`

`src/server/studentAccessCodes.functions.ts`

Server functions TanStack Start avec service role :

- `listClassStudentAccess(input)` : retourne classe + eleves + statut code actif/revoque ;
- `generateClassStudentAccessCodes(input)` : genere ou regenere les codes pour une classe, retourne les codes en clair uniquement dans la reponse ;
- `revokeStudentAccessCode(input)` : revoque le code actif d'un eleve ;
- `generateSingleStudentAccessCode(input)` : regenere un seul eleve.

Securite :

- `auth.getUser(accessToken)` en premier ;
- profil appelant obligatoire ;
- permissions : `superadmin`, `admin`, `ddfpt`, `principal` ;
- superadmin peut agir cross-tenant ;
- admin/ddfpt/principal limites a leur `establishment_id` ;
- audit log pour generation classe, generation individuelle, revocation.

Hash :

- code clair genere cote serveur ;
- hash SHA-256 avec salt env `STUDENT_ACCESS_CODE_PEPPER` si present ;
- fallback sans pepper autorise en dev mais rapport doit le signaler ;
- code clair jamais stocke.
