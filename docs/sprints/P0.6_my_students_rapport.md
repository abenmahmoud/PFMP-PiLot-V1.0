# P0.6 — Rapport Codex — My students Supabase

## Fichiers modifies / crees

- `src/services/myStudents.ts` : service de lecture Supabase pour les eleves d'un referent/principal.
- `src/routes/my-students.tsx` : route dual-mode demo/Supabase, empty states, filtres, cartes mobile-first.
- `docs/sprints/P0.6_my_students_plan.md` : plan d'execution.
- `docs/sprints/P0.6_my_students_rapport.md` : rapport.

## Decisions techniques

- Le mode demo conserve l'ancien `RoleGuard` + `StudentCard`.
- Le mode Supabase utilise `useAuth()` et n'importe plus `useCurrentUser()`.
- La RLS n'est pas contournee : aucune requete ne force `establishment_id`.
- Les relations sont chargees en batch pour eviter le N+1.
- Les boutons de carte sont utilisables sur mobile avec des zones d'action de hauteur minimale 44px.

## Ce qui reste mocke / TODO

- Les tests RLS automatises Prof A / Prof B restent a executer cote Supabase quand le MCP sera stable.
- `StudentCard` reste demo-only pour l'instant; la carte Supabase est locale a `/my-students`.
