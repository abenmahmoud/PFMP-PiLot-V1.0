## Précisions et simplifications proposées

### ✅ Sur ton étape 3 — `useCurrentUser`

**Tu peux passer cette étape quasiment en mode "audit only".** J'ai
vérifié le code de `src/lib/useCurrentUser.ts` avant de t'envoyer le
brief : le bridge démo↔Supabase est **déjà fonctionnel** depuis le
Sprint 2 polish. Voici le code actuel pour confirmation :

```ts
export function useCurrentUser(): Profile {
  const auth = useAuth()
  const demoId = useSyncExternalStore(subscribe, getStoredId, () => CURRENT_USER_ID)

  if (isDemoMode() || !auth.profile) {
    return (
      profiles.find((p) => p.id === demoId) ||
      profiles.find((p) => p.id === CURRENT_USER_ID)!
    )
  }

  return profileRowToProfile(auth.profile)
}
```

**Comportement :**
- `isDemoMode() === true` → user démo (switcher rôle marche)
- `!auth.profile` (= prod, pas loggé) → user démo de fallback
- Sinon → profile Supabase mappé

**Le seul trou potentiel** : en prod vide, si `!auth.profile` retourne
le user démo de fallback, on a un comportement incohérent (l'app
pense que tu es Julien Garcia alors que t'es pas loggé). Mais
`AuthProvider` + redirect `/login` gèrent normalement ça avant.

→ **Action concrète** : confirme dans ton rapport ce que tu observes
en chargeant l'app sans Supabase configuré + sans `VITE_DEMO_MODE`.
Si ça redirect bien vers `/login`, RAS. Si ça crashe, patch minimal.
