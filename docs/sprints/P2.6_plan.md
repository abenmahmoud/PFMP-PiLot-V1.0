# P2.6 - Studio conventions numeriques par classe

## Objectif

Mettre en place le socle durable qui permet d'affecter une convention numerique a chaque classe, puis de generer automatiquement les conventions nominatives des eleves d'une periode PFMP.

## Logique metier cible

1. Le superadmin ou le DDFPT cree/importe des modeles de convention officiels.
2. Un modele de convention est affecte a une classe.
3. Quand une periode PFMP est creee ou synchronisee pour cette classe, chaque eleve obtient un dossier PFMP et une convention logique.
4. La convention utilise le modele affecte a la classe.
5. La convention peut suivre deux circuits :
   - numerique : generation PDF, signature entreprise/parent/etablissement, archivage ;
   - papier : generation/impression, scan signe, televersement, archivage.

## Scope de ce commit

- Ajouter une table d'affectation classe -> modele de convention.
- Etendre les types locaux.
- Ajouter les server functions de lecture/affectation.
- Modifier la preparation des conventions pour choisir le modele de la classe.
- Ajouter une interface "Studio conventions" dans la page Documents.
- Retirer la logique de bouton mort : les actions non encore implementees sont presentees comme etapes a venir, pas comme boutons actifs.

## Hors scope volontaire

- Import DOCX complet avec preservation de mise en page Word.
- Generation PDF finale signee.
- Upload papier signe.
- Portail eleve "j'ai trouve un stage".

Ces elements seront branches au-dessus de ce socle, sans changer la structure d'affectation.

## Contraintes

- Additif seulement cote SQL.
- Tenant strict.
- DDFPT = admin document du lycee.
- safeHandlerCall sur server functions.
- Typecheck et build verts.

## Validation attendue

- Un DDFPT voit les modeles de convention et les classes du tenant.
- Il affecte un modele a une classe.
- La synchronisation d'une periode cree/met a jour les conventions avec le template de la classe.
- La page Documents donne une vision claire : modeles, affectations, conventions generees/manquantes.
