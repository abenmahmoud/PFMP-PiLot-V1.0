# P1.3 — Plan d'execution Codex — Import SIECLE classes + eleves

Date : 12 mai 2026
Branche : `feature/p1-3-siecle-import`
Objectif : permettre a un admin, DDFPT ou superadmin d'importer un export SIECLE CSV/XLS/XLSX pour creer les classes et eleves du tenant actif, sans creer de comptes eleves.

## Verification initiale

- `main` a ete synchronise apres merge P1.2.4.
- La branche de travail `feature/p1-3-siecle-import` a ete creee depuis `main`.
- `xlsx` n'est pas encore present dans `package.json`, contrairement a l'hypothese du brief. Je vais l'ajouter explicitement pour respecter le support XLS/XLSX, et le justifier dans le rapport.
- Aucune migration SQL ne sera ajoutee.

## Ordre d'implementation

1. Ajouter le seed fictif RGPD-safe dans `docs/sprints/seed/siecle_eleves_fictifs.csv`.
2. Creer le parseur pur `src/services/imports/siecle.ts`.
   - Parsing client-side uniquement.
   - CSV avec separateur `;` ou `,`.
   - XLS/XLSX via `xlsx`.
   - Normalisation des en-tetes SIECLE.
   - Dedoublonnage par `nom + prenom + division`.
   - Erreurs non bloquantes dans `errors[]`.
3. Creer `src/server/imports.functions.ts`.
   - `createServerFn({ method: 'POST' })`.
   - Verification `auth.getUser(accessToken)`.
   - Controle permissions admin/DDFPT/superadmin.
   - `dryRun: true` sans aucune ecriture.
   - Commit reel avec creation/reutilisation classes, insertion eleves, detection doublons DB.
   - Audit log `import.siecle.students`.
4. Creer la route `src/routes/imports.siecle.tsx`.
   - Workflow 4 etapes : upload, preview, selection divisions, confirmation.
   - Max fichier 5 MB.
   - Aucun stockage du fichier.
   - Mode demo simule sans ecriture Supabase.
5. Ajouter l'entree Sidebar `Import SIECLE` pour `admin`, `ddfpt`, `superadmin`.
6. Ecrire `docs/sprints/P1.3_rapport.md`.
7. Valider avec `npm run typecheck`, `npm run build`, puis `git diff --check`.

## Points de securite

- Le client ne decide pas la securite : la server function verifie le caller et injecte l'`establishment_id`.
- Aucun email eleve n'est envoye.
- Aucun utilisateur Supabase Auth n'est cree pour les eleves.
- Le fichier importe reste en memoire navigateur et n'est jamais envoye ni stocke.
- La service-role key reste cote server function uniquement.

## Strategie doublons

- Au parsing : dedoublonnage exact normalise sur `lastName + firstName + divisionName`.
- En base : skip si un eleve actif existe deja avec meme `first_name`, `last_name` et `class_id`.
- Dans un meme import : skip si le meme eleve apparait deux fois pour la meme classe.

## Definition de pret

- Route `/imports/siecle` visible dans la sidebar pour les bons roles.
- Dry-run obligatoire avant import reel.
- Import reel cree les classes manquantes et les eleves associes.
- Les eleves apparaissent ensuite dans `/students`.
- Rapport P1.3 complet avec limites et preuves de validation.
